# YeMind Zen 开发计划

YeMind Zen 采用“小而精、上游优先”的开发方式：每次只开发一个主题；先完整阅读 `simple-mind-map` 对应插件、命令、事件和数据结构；能直接调用原生能力时，不再增加第二套实现。

## v0.5.2：上游优先稳定性（已完成）

- 移除自制拖拽视图控制器
- 固定使用原生结构化子树拖拽
- 恢复标签页等待仓库初始化
- 串行化导图数据保存
- 关闭标签页前刷新待保存内容

## v0.5.3：原生复制、剪切与粘贴（已完成）

- 直接调用 `Render.copy`、`Render.cut` 和 `Render.paste`
- 使用原生内存剪贴板，不依赖浏览器剪贴板权限
- 完整复制节点子树
- 多选时只保留最高层节点，避免重复子树
- 使用原生命令处理剪切、粘贴、新 ID、激活、历史和重新布局
- 保留上游 `KeyCommand` 对 Ctrl/Cmd+C/X/V 的唯一所有权
- 当前范围限定为同一张导图，不实现跨导图粘贴

## v0.5.4：只做框选与多选稳定性

开发前完整分析 `Select`、`Drag`、激活节点列表和多节点命令，只实现：

- Ctrl/Cmd 多选
- 空白画布框选
- 多节点批量结构拖动
- 画布边缘移动和取消操作
- 与右键菜单、删除、复制和剪切的联动

本版不同时开发外框、概要增强或跨导图粘贴。

## v0.5.5：只做概要与关联线完善

- 原生 Generalization 和 AssociativeLine 数据结构
- 多节点概要范围
- 关联线文字、端点与控制点
- 删除节点时关联数据清理

## v0.6.0：格式兼容与导出

按格式逐个交付，不在一个版本同时实现所有格式。优先顺序：Markdown → PNG/SVG → XMind → FreeMind → KMindz。

## v0.7.0：思源深度集成

按“节点子文档 → 文档拖入 → 块拖入 → 镜像块 → 文档树导图”的顺序逐项开发。
