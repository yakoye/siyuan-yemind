# YeMind Zen 开发计划

YeMind Zen 采用“小而精、上游优先”的开发方式：每次只开发一个主题；先完整阅读 `simple-mind-map` 对应插件、命令、事件和数据结构；能直接调用原生能力时，不再增加第二套实现。

## v0.5.2：上游优先稳定性（已完成）

- 移除自制拖拽视图控制器
- 固定使用原生结构化子树拖拽
- 恢复标签页等待仓库初始化
- 串行化导图数据保存
- 关闭标签页前刷新待保存内容

## v0.5.3：原生复制、剪切与粘贴（已完成）

- 直接调用 `Render.copy`、`Render.cut` 和 `Render.paste`
- 使用原生内存剪贴板，不依赖浏览器剪贴板权限
- 完整复制节点子树
- 多选时只保留最高层节点，避免重复子树
- 使用原生命令处理剪切、粘贴、新 ID、激活、历史和重新布局
- 保留上游 `KeyCommand` 对 Ctrl/Cmd+C/X/V 的唯一所有权
- 当前范围限定为同一张导图，不实现跨导图粘贴

## v0.5.4：原生框选与多节点批量移动（已完成）

- 完整分析并固定 `Select`、`MindMapNode`、`Drag` 的上游行为
- 平移优先：左键拖动画布，Ctrl/Cmd + 左键框选
- 选择优先：左键框选，右键拖动画布
- Ctrl/Cmd + 单击增减节点选择
- 拖动任一已选节点时，由原生 Drag 批量移动最上层所选子树
- 增加轻量模式切换和多选数量提示，不新增第二套选择状态
- 与原生复制、剪切、删除和历史记录共享同一激活节点列表

## v0.5.5：原生概要与关联线完善（已完成）

- 多节点概要范围继续使用原生 `parseAddGeneralizationNodeList` 和 `ADD_GENERALIZATION`
- 区分添加概要、删除当前概要、删除所属节点全部概要
- 关联线创建、文字编辑、删除、端点和控制点继续由 `AssociativeLine` 插件负责
- 增加轻量创建中/已选中状态栏，不维护第二套关联线状态
- 仅增加默认概要文字、默认关联线文字、层级和控制点四项原生设置
- 在持久化边界清理已删除目标和重复目标，保持原生并行数组对齐

## v0.5.6：现有功能回归与稳定性修复（已完成）

- 不增加新功能，完整回归右键菜单、文字选区、节点与画布工具栏
- 检查保存、新建、重命名、删除、关闭、恢复标签页和设置写盘失败路径
- 检查结构化拖拽、框选、多选、缩放、适配视图、重置视图和布局
- 修复只读/根节点保护、多选右键主节点、持久化事务、链接跳转和图片加载错误
- 保持拖拽、选择、缩放、布局、历史、概要和关联线由上游内核负责

## v0.5.7：只做外框与多节点外框

- 先完整分析上游 OuterFrame 插件、节点数据、选择范围、历史和删除行为
- 单节点外框与多节点外框
- 外框文字与基础样式
- 删除节点时外框范围更新
- 不同时加入检查点、导出或思源集成

## v0.6.0：格式兼容与导出

按格式逐个交付，不在一个版本同时实现所有格式。优先顺序：Markdown → PNG/SVG → XMind → FreeMind → KMindz。

## v0.7.0：思源深度集成

按“节点子文档 → 文档拖入 → 块拖入 → 镜像块 → 文档树导图”的顺序逐项开发。
