# YeMind Zen 开发计划

YeMind Zen 采用“小而精、上游优先”的开发方式：每次只开发一个主题；先完整阅读 `simple-mind-map` 对应插件、命令、事件和数据结构；能直接调用原生能力时，不再增加第二套实现。

## v0.5.2：上游优先稳定性（已完成）

- 移除自制拖拽视图控制器
- 固定使用原生结构化子树拖拽
- 移除自由拖拽和拖拽后视图恢复实验设置
- 恢复标签页等待仓库初始化
- 串行化导图数据保存
- 关闭标签页前刷新待保存内容
- 为上述生命周期增加专门回归测试

## v0.5.3：只做复制、剪切与粘贴

开发前完整分析 `Render.copyNode`、`cutNode`、`pasteNode`、Clipboard 和多选节点过滤。只实现：

- 同一导图复制、剪切、粘贴
- 完整子树复制
- 多选时只复制最顶层节点，避免重复子树
- 粘贴后的激活、历史记录和自动保存
- 关闭剪贴板权限时的降级路径

本版不同时开发外框、概要增强或跨导图粘贴。

## v0.5.4：只做框选与多选稳定性

- 原生 Select 插件行为分析
- Ctrl/Cmd 多选和框选
- 多节点批量移动
- 画布边缘移动验证
- 与节点右键菜单、删除和复制的联动

## v0.5.5：只做概要与关联线完善

- 原生 Generalization 和 AssociativeLine 数据结构
- 多节点概要范围
- 关联线文字、端点与控制点
- 删除节点时关联数据清理

## v0.6.0：格式兼容与导出

按格式逐个交付，不在一个版本同时实现所有格式。优先顺序：Markdown → PNG/SVG → XMind → FreeMind → KMindz。

## v0.7.0：思源深度集成

按“节点子文档 → 文档拖入 → 块拖入 → 镜像块 → 文档树导图”的顺序逐项开发。
