# YeMind Zen v0.5.18 官方大纲编辑与拖拽研究

日期：2026-07-18  
官方参考：用户提供的 KMind Zen 包 `siyuan-kmind-zen-v0.3.4.zip`，包内版本为 0.34.0。

## 研究目标

本次只研究分屏/纯大纲的编辑生命周期、结构拖拽、折叠状态和显示层，不引入与本轮无关的官方模块。

## 官方实现特征

KMind Zen 0.34.0 的大纲建立在连续 ProseMirror/Tiptap 文档上。节点行是稳定块实体，文字输入只产生编辑器事务，不会在每个字符提交后把活动行从 DOM 中移走。结构拖拽使用指针位置、行几何和层级信息计算插入槽，再通过统一命令更新文档。

这带来四个可移植原则：

1. 活动编辑器 DOM 在普通文字同步期间必须稳定。
2. 文字事务和结构事务必须分开，结构事务前只提交一次草稿。
3. 拖拽不能只看纵向位置，还要把横向缩进解释为父子层级意图。
4. 折叠是独立结构状态，不能依赖旧 DOM 状态或未完成的异步焦点任务。

## 为什么不完整复制官方大纲

官方连续大纲依赖其文档 schema、命令总线、历史、节点实体、资产和同步层。直接复制会让 YeMind 同时拥有官方文档树和 `simple-mind-map` 树，产生第二套结构、撤销历史和持久化格式。

思源自带大纲同样围绕思源块 ID、Protyle 和内核事务工作，不能直接把其 DOM 或内部命令绑定到导图 UID。把外观复制过来不能自动获得其稳定事务语义。

因此本版继续采用适配方案：移植经过验证的交互和事务原则，最终节点结构仍由当前 `simple-mind-map` 命令执行。

## v0.5.17 根因复现

### Delete/Backspace 后退出编辑

旧 `patchOutlineTree()` 将所有现有行依次加入 `DocumentFragment`，最后重新追加到大纲容器。即便行对象没有重建，这个过程仍会把活动 contenteditable/Quill 行从父容器移出再插入，浏览器随之丢失真实焦点和 selection。

Quill 在输入后延迟提交文字；上游 `SET_NODE_TEXT` 同步触发 `data_change`；大纲 patch 移动活动行。因此用户看到 Delete 只删除一个字符后退出，Backspace 也可能失效。

### 拖动后出现空白小圆圈

拖动前 `flush()`、同步 `data_change`、大纲 render 和 detach 可能重入。若活动 Quill 已被 DOM patch 移动，后续流程可能读取到空编辑层并把空 HTML 写回。画布节点没有文字后只剩极小轮廓。

### 折叠后不能展开

折叠按钮、行点击、编辑器 detach 和异步焦点恢复共用一条路径。折叠时被隐藏的活动子节点还可能留下永远无法完成的 pending focus，影响下一次 render 和按钮操作。

## v0.5.18 适配

### 不移动活动行的 keyed patch

新 patch 按 UID 复用行，只在真实顺序变化时对相关行调用 `insertBefore()`。顺序未变化时，活动行不从 DOM 中移出；模型文字回流也不覆盖活动 Quill 内容。

新增结构签名只包含 UID、深度、子节点存在性和展开状态。普通文字或富文本变化不会被误判为结构变化。

### 提交防重入

`OutlineRichTextController` 增加 committing guard。同步模型回调再次要求 flush 时直接返回；一次用户变化最多形成一次上游文字提交。结构操作使用带原因的 `commitAndDetach()`，避免重复 flush。

### 整行 Pointer 拖拽

删除六点按钮和 HTML5 `draggable`：

- 行非正文区域移动超过阈值后立即拖拽；
- 正文内先保留光标和文字选择，长按后移动才进入拖拽；
- 按钮、链接和其他交互控件不启动拖拽；
- 自绘 ghost 和 before/inside/after 提示；
- 横向位置按 22px 层级宽度解析同级、子级或向左降低层级；
- 自身和后代目标被拒绝；
- 最终仍调用上游 `MOVE_NODE_TO`、`INSERT_BEFORE`、`INSERT_AFTER`。

### 折叠状态

按钮从当前最新 `expanded` 状态计算下一状态，并阻止事件穿透。叶节点使用不可交互占位，不再显示小圆点按钮。若折叠使活动子节点不可见，焦点恢复任务会被取消；若活动行仍可见，则按原 selection 恢复。

### π 与模糊

公式按钮使用独立数学字体 span 和辅助标签，不把视觉字符写入导图内容。模糊继续兼容旧 cloze 数据，但显示层改为画布/大纲共用的 `filter: blur()`、半透明背景和 `backdrop-filter`；悬停恢复内容。

## 保持不变的边界

- 不增加第二套导图树。
- 不增加第二套撤销历史。
- 不复制 KMind 私有 ProseMirror schema 和命令总线。
- 不直接改写节点树；最终结构仍由上游命令完成。
- 不改变导图、检查点和备注/批注存储格式。
