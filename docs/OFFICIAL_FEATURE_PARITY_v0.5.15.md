# YeMind Zen 与 KMind Zen 官方功能对照（v0.5.15）

本表以用户提供的 KMind Zen 0.33.0 运行源码为参考。v0.5.15 专门对照选区富文本工具栏、持续编辑器和弹层挂载机制；用户明确要求的颜色色板、默认重置和吸管属于在官方能力基础上的交互改进。原则仍是：复用 `simple-mind-map` 的结构、历史和持久化，独立移植可验证的交互机制，不嵌入 KMind 私有文档内核。

| 功能域 | KMind Zen 官方实现 | YeMind Zen v0.5.15 | 状态 |
|---|---|---|---|
| 宿主弹层隔离 | 编辑器局部 portal/overlay 层 | 所有 YeMind 浮层挂载 `.ymz-editor`，局部 stacking context | 已对齐目标 |
| 节点文字编辑 | 持续富文本编辑器，选区驱动 | 上游 RichText，编辑文字/光标继承节点样式 | 已修复不可见问题 |
| 局部文字选区 | 可选节点中的部分文字 | 关闭进入编辑时整段全选，保留自然 selection | 已完成 |
| 选区工具栏 | 节点与大纲共用丰富格式工具 | `RichTextFormattingTarget` 统一导图与大纲 | 已完成当前操作集 |
| 文字/背景颜色 | 官方颜色字段与清除入口 | 直观色板 + 重置默认 + 更多颜色 + EyeDropper | 按用户要求改进 |
| 分屏/大纲富文本 | 持续文档编辑器 | 单活动 Quill 行编辑器，其余行为轻量 HTML | 已完成性能适配 |
| 大纲富文本保存 | 官方文档事务 | 上游 `SET_NODE_TEXT(..., richText=true)` | 已上游化 |
| 结构、历史、持久化 | KMind 私有内核 | `simple-mind-map` 与 YeMind 仓库/检查点 | 保持原实现 |
| 默认父子连线 | Cubic 平滑曲线 | `simple-mind-map` `curve` | 已对齐 |
| 主题 | 13 个官方主题家族 | 13 个同名明暗预设 | 已完成首版适配 |
| 节点拖动 | 帧合并、候选稳定、贝塞尔提示 | 官方矩形/尾区/通道与 60 ms/3 帧机制 | 主要手感已对齐 |
| 可编辑大纲结构行为 | 事务提交、光标恢复 | IME 安全事务与 UID selection 恢复 | 已对齐核心 |
| 节点整体样式 Inspector | 完整节点/作用范围面板 | 当前已有选区文字格式，完整节点面板待补 | 后续 |
| 高级边线路由 | 多种私有 renderer | 弧线、圆角折线、直线 | 后续独立适配 |
| 导入导出 | Markdown、图片、KMind 等 | 基础上游 Export 已注册，完整格式 UI 待补 | 后续 |

## v0.5.15 边界

- 没有把整个 KMind Zen 打包运行时放入插件。
- 没有增加第二套树结构、撤销历史、检查点或导图仓库。
- 大纲只创建一个活动 Quill 实例，避免为每个节点长期维护编辑器。
- HTML 写回前保持现有富文本格式注册和安全清理边界。
- `EyeDropper` 不受所有浏览器内核支持；不可用时自动回退到系统颜色选择器。
