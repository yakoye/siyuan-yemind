# YeMind v0.6.4 官方交互适配边界

日期：2026-07-20

## 本版范围

v0.6.4 继续采用“移植交互与事务机制、保留 YeMind 数据和历史边界”的方案，完成查找/替换、大纲叶子标识、拖拽热区、思源搜索结果扩展和整图样式。

## 已适配

- 导图内搜索默认仅显示查找行，展开按钮显式切换替换行。
- 大纲和分屏使用小圆点标识非 Root、非概要的叶子节点；当前行沿用主题强调色。
- 大纲结构拖拽只能从节点文字左侧空白区开始；正文区域始终优先进入文字编辑、放置光标或拖选文字。
- 思源搜索输入时，YeMind 扫描导图标题、节点文字、备注、批注、标签、链接和待办，并把独立结果区挂载到当前搜索界面；点击结果打开对应导图并定位节点。
- 顶部“样式”和空白画布菜单控制整张导图的密度、彩虹连线和背景色。
- 节点右键菜单中的“节点样式”继续只处理当前选中节点，两套样式入口使用不同图标、名称和数据范围。
- 整图样式进入 YeMind 文档、检查点、保存和重新打开流程。

## 保持不变

- 不引入 KMind 私有文档模型、React 状态树或第二套撤销历史。
- 不替换 `simple-mind-map` 的节点结构、布局、拖拽落点、节点样式或历史命令。
- 不把叶子圆点写入节点数据；它只是大纲的结构提示。
- 不修改思源数据库或宿主全文索引文件；YeMind 结果通过插件监听当前搜索输入并扩展搜索界面。
- 不把查找面板的展开状态写入导图内容。

## 自动验证边界

自动测试覆盖查找/替换状态、叶子判定、拖拽热区、全局匹配内容、搜索结果挂载与打开回调、整图样式事务、检查点和持久化。真实 Windows SiYuan 3.7.2 中的搜索 DOM、插件结果区位置、宿主主题、不同显示缩放和大型导图搜索耗时仍需桌面验收。
