# YeMind v0.5.17 官方大纲、备注与图片交互研究

日期：2026-07-18

## 参考范围

本版以用户提供的 `siyuan-kmind-zen-v0.3.4.zip` 为最新官方参考。包名中的 `v0.3.4` 与内部版本不同，`plugin.json` 和官方 README 均标记为 **0.34.0**。

官方 0.34.0 明确新增或修正：

- 节点正文和大纲行图片粘贴，并可拖动调整图片大小；
- 备注区图片粘贴和备注框尺寸拖动；
- 根节点大纲行按 Enter 创建子节点；
- 备注悬浮窗布局与保存按钮；
- 0.33.0 已处理大纲非法内容初始化和展开卡死等稳定性问题。

## 官方机制结论

官方顺滑的大纲不是“一行一个临时 textarea + 每次数据变化整表重画”。从运行包可确认其交互围绕以下机制组织：

1. 行实体具有稳定身份和层级/排序信息，普通正文提交不会令当前编辑行失去身份。
2. 文字编辑与结构拖拽是两类事务；结构动作前先提交或 flush 当前正文。
3. 拖拽使用独立行 chrome/handle/drop slot，避免与正文选区争抢鼠标。
4. 根节点、普通节点和只读状态具有明确的 Enter、拖拽和修改边界。
5. 备注是独立长文内容面，不等同于按时间追加的批注。
6. 节点正文、大纲正文和备注图片都保存自然尺寸/比例信息，再由编辑器提供拖动调整。

## YeMind 采用的边界

YeMind 继续采用“适配交互机制，不导入官方私有内核”的方案：

- 唯一正式树仍是 `simple-mind-map` 数据；
- 唯一结构历史仍由 `simple-mind-map` 命令栈维护；
- 唯一正式导图库、检查点和自动保存仍由 YeMind 仓库维护；
- 大纲只增加稳定 UID DOM patch、单活动 Quill 和结构事务适配；
- 行拖拽只计算 before/inside/after 意图，最终调用 `INSERT_BEFORE`、`MOVE_NODE_TO`、`INSERT_AFTER`；
- 备注使用独立 `yemindNote` 字段，不混入 `yemindComments` 时间线。

## v0.5.17 的具体适配

### 稳定大纲

- `flattenOutline` 生成稳定 UID 行；
- `patchOutlineTree` 复用已有行 DOM，当前活动 UID 的编辑器内容不被模型回流覆盖；
- 一个 `OutlineRichTextController` 只维护一个活动 Quill；
- IME 组合期不自动提交、不执行结构快捷键；
- 空正文也是合法提交，删除文字与删除节点完全分离；
- 同步结构命令执行前预置待恢复焦点，避免上游同步触发 `data_change` 时丢失光标。

### 行级拖拽

- 每行显示独立拖拽把手；
- 目标上 25% 为之前、中间 50% 为子节点、下 25% 为之后；
- 禁止根节点移动、拖入自身或后代、只读拖拽；
- 结构变更仍由上游命令完成，可进入现有撤销/重做。

### 备注与批注

- 备注允许长文、粘贴图片和保存窗口宽高；
- 备注 HTML 在保存和展示边界清理脚本、事件属性和危险 URL；
- 备注与批注分别显示图标，均支持延时悬停预览；
- 批注仍保留多条时间线语义，不与备注字段合并。

### 颜色与焦点

- 色板之后显示“重置默认/更多颜色”和可编辑 HEX/RGB；
- 合法值实时双向转换并应用；
- 不完整值不写入格式；
- 输入、粘贴、Backspace、IME 等事件在颜色浮层内终止传播；
- 打开浮层前的富文本 selection 可恢复，Escape 回退打开前的颜色。

## 未导入的官方能力

本版没有导入官方多根文档模型、ProseMirror 文档内核、私有历史、跨画布/大纲双向拖拽协议、`.kmindz.svg` 文件格式、Pro 授权能力或思源文档树/MOC 集成。
