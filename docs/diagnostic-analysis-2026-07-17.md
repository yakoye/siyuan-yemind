# YeMind Zen 真实诊断包分析（2026-07-17）

## 结论

用户的三张真实导图没有被诊断功能删除。第二份诊断包的完整内容附件中仍包含三张导图，两份日志中均没有真实的 `map-deleted` 事件。

“运行诊断后出现第三张，随后三个中消失一个”的直接原因，是 v0.5.9 生命周期自检把临时测试导图创建在正式 `MapRepository` 中。Dock 收到正式仓库更新后显示临时导图，自检结束又在 `finally` 中删除，因此产生了真实文件被删除的视觉错觉。

同时确认了两个独立问题：

1. 切换纯大纲后，隐藏画布仍调用 `resize()`，产生 `容器元素el的宽高不能为0`。
2. 第三张导图收到多次打开请求，但没有编辑器挂载，说明恢复标签容器或标签句柄处于无效状态。

## 修复策略

- 生命周期自检迁移到独立临时文件和独立仓库。
- Dock 等待正式仓库加载完成。
- 标签等待容器宽高有效后再创建导图库实例。
- 大纲模式不再调用隐藏画布 resize。
- 陈旧标签句柄自动失效，允许重新打开。
- 待办复选框只负责勾选和取消勾选。
- 拖拽继承上游 Drag，只增加目标父节点虚线。
