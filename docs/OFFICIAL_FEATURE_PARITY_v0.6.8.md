# YeMind v0.6.8 官方功能适配边界

## 适配目标

本版参考思源原生全局搜索的三段式结构：顶部查询条件、左侧结果列表、右侧/下方内容预览。YeMind 不再创建独立悬挂结果卡片，而是适配宿主现有 DOM 与键盘习惯。

## 已适配

- 使用 `#searchList` 作为唯一结果挂载点。
- 使用 `b3-list-item`、`b3-list-item__text`、`b3-list-item__meta` 等宿主视觉结构。
- 使用 `#searchPreview` 显示只读大纲覆盖层。
- 保留思源原生预览 DOM，切换普通结果时直接恢复。
- 支持单击预览、双击/Enter 打开、Alt 操作右侧打开、方向键与翻页键。
- 结果列表和预览被宿主重建时自动恢复。

## 未复制的宿主私有实现

- 未调用思源内部未公开的搜索结果状态机或私有 Protyle 构造器。
- 未伪造思源块 ID，也未把导图节点写入思源数据库。
- YeMind 条目视觉上遵循原生结果，但使用插件自己的选中与预览状态，避免宿主把导图 UID 当作块 ID。
- 预览使用只读大纲，不在搜索窗口中启动完整 `simple-mind-map` 编辑器。

## 降级策略

- 节点定位失败时仍打开对应导图。
- 已经打开的导图标签优先复用，不重复创建编辑器实例。
- 宿主结构缺少标准 `#searchPreview` 时创建插件局部只读预览容器。
