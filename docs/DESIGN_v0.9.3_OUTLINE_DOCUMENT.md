# YeMind v0.9.3 continuous outline design

## Problem statement

The previous outline was a list of independent rich-text editors. That model is appropriate for precise formatting of one node, but the browser cannot create one native text selection across several independent editors. As a result, users could not select, copy, cut, replace or paste a whole section like a normal text document.

v0.9.3 treats this as an editing-model problem rather than a clipboard patch.

## Reference interaction model

The design follows the common document-outline model used by code editors and plain-text outliners:

- one continuous text buffer;
- one node per line;
- leading indentation represents parent-child depth;
- native browser selection and clipboard behavior span any number of lines;
- Tab and Shift+Tab indent or outdent every selected line;
- Enter continues the current line indentation;
- a secondary node-tree mode remains available for rich-text formatting, node dragging and explicit expand/collapse operations.

This is intentionally a dual representation of one tree, not two independent data stores.

## Tree-to-text contract

- Four spaces are emitted for each hierarchy level.
- Rich-text node labels are converted to visible plain text in document mode.
- Stable node UIDs and non-text metadata are preserved when a line can be reconciled with an existing node.
- A changed line becomes plain text while retaining its UID and other metadata; structural-path reconciliation protects even a completely rewritten label.
- New lines receive new UIDs.

## Text-to-tree contract

- Spaces, tabs and copied non-breaking spaces are accepted.
- The indentation width is inferred from the pasted document.
- An impossible depth jump is clamped to one level deeper than the preceding line.
- Common copied table-of-contents escapes such as `\:` and `\;` are normalized.
- Exactly one zero-indent first line is treated as the document root.
- When several lines genuinely start at zero indentation, the existing center topic is preserved and those lines become its first-level children.
- The complete result is committed through one upstream `updateData()` transaction, preserving undo history. `setData()` is deliberately not used because it clears history.

## Synchronization and ownership

- Text input is reconciled after a short debounce or immediately on blur, Ctrl/Cmd+Enter and Ctrl/Cmd+S.
- IME composition is held as an editor-local draft and is never reconciled mid-composition.
- Before the canvas takes editing ownership, pending outline text is flushed.
- A map event generated by the text transaction does not overwrite the active textarea selection.
- Switching to node-tree mode flushes the text document first.
- Switching back serializes the current map into the same continuous text document.
- Readonly mode applies to both representations.

## Hover action contract

The node action buttons belong to a pointer interaction region formed by the SVG node, a short invisible bridge and the HTML action host. Leaving the SVG node starts a delayed close; entering the bridge or buttons cancels it. Selection continues to pin actions, but selection is no longer required.

## Root background contract

A transparent center-node fill is resolved against the effective canvas background after project overrides. Explicit center fills and node-local fills still take precedence. This keeps branch lines from showing through center-topic text without changing theme source data.
