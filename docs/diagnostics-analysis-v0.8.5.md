# YeMind v0.8.5 用户诊断包分析

## 诊断包

- 插件版本：0.8.4
- 事件数量：209
- 导图结构自检：通过

## 关键事件

诊断中多次出现以下链路：

1. `rich-text / opened`：左侧画布新节点开始编辑；
2. `editor / data-change`：新节点插入触发结构变化；
3. `outline / editor-destroy`，原因 `structure-change`；
4. 紧接着 `outline / edit-start`：旧右侧大纲节点被重新激活。

在切换到分屏后的最后一次复现中，序列 200—204 完整出现该链路，证明跳转不是画布新节点命令选择了错误节点，而是大纲重绘阶段主动恢复了旧编辑器。

## 根因

`renderOutline()` 在任何结构变化中，只要检测到活动大纲编辑器，就保存其选区并创建待恢复焦点。它没有判断结构变化来自大纲还是画布，因此画布新增节点也会复活旧大纲节点。

## 修复

v0.8.5 使用编辑表面所有权和显式票据：

- 画布接管后旧大纲编辑器提交并关闭；
- 旧恢复票据失效；
- 普通结构变化不再推断焦点；
- 只有大纲自身结构命令可以请求恢复。
