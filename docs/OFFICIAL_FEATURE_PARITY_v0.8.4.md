# YeMind v0.8.4 拖动后富文本编辑适配边界

日期：2026-07-21

## 上游行为

YeMind 继续使用 `simple-mind-map` 的 SVG 节点、结构拖动、Quill 富文本编辑器、节点 UID、历史命令和 `SET_NODE_TEXT` 保存路径。

结构拖动会触发节点重新布局或重新渲染。富文本编辑开始后，上游还会暂时隐藏原 SVG 文字，并调用 `updateTextEditNode()` 更新编辑层位置。在思源标签页的嵌套定位环境中，这两个行为可能同时造成：

- 编辑事务仍保存拖动前的渲染节点对象；
- 隐藏或失效的 SVG 文字返回 `0 × 0` 的 `getBoundingClientRect()`；
- 上游把零矩形继续换算成编辑框位置，产生 `-6px / -4px`；
- 编辑结束时可能向拖动前的节点实例提交内容。

## v0.8.4 适配

### 1. 稳定节点身份

富文本事务开始时保存节点 UID。每次显示、重新定位、聚焦和提交前，都通过 `renderer.findNodeByUid(uid)` 获取当前渲染实例。

这意味着拖动、同位拖动或结构重绘替换了 SVG 节点对象后，编辑器仍绑定到同一个逻辑节点，而不是继续使用失效对象。

### 2. 有效几何规则

编辑定位不再信任任意 `getBoundingClientRect()` 结果。矩形必须满足：

- 坐标与尺寸均为有限数值；
- 宽度和高度大于最小有效阈值；
- SVG 元素仍连接在当前文档中。

定位回退顺序为：

1. 当前节点有效的 DOM 视口矩形；
2. 当前 SVG 的 `getScreenCTM()` 与文本尺寸重建的屏幕矩形；
3. 本次编辑事务最后一次有效锚点。

零尺寸、脱离 DOM 或非有限矩形不得覆盖已经有效的编辑位置。本版没有针对 `-6/-4` 写常量修正。

### 3. 当前节点提交

结束编辑前再次按 UID 解析当前渲染节点，再调用上游保存流程。因此拖动后输入的文字写入当前节点及其历史记录，不会写入拖动前的失效实例。

### 4. 保留既有编辑语义

本版不改变 v0.8.3 已建立的行为：

- 新节点和默认节点首次双击自动全选；
- 已编辑节点双击后光标位于末尾；
- Ctrl/Cmd+A、复制、剪切、粘贴、Backspace、Delete 和中文输入法仍由 Quill 处理；
- 分屏和大纲继续使用原有稳定 keyed patch 与单活动编辑器。

### 5. 诊断增强

富文本诊断事件新增：

- 原始 SVG 矩形宽高；
- 最后有效锚点的位置和尺寸；
- SVG 是否仍连接文档；
- 稳定节点 UID 是否存在；
- 本次位置来自 DOM 矩形、SVG 屏幕矩阵还是最后有效锚点；
- 无有效位置时的跳过事件。

## 不改变的兼容边界

- 当前插件 ID：`siyuan-yemind`；
- 历史协议链接别名：`siyuan-yemind-zen`；
- 导图、设置和检查点数据格式不升级；
- 不修改 `simple-mind-map` 包源码；
- 不建立第二套富文本数据模型；
- 节点结构拖动和撤销历史仍由上游命令管理。
