# YeMind Zen v0.5.2 Upstream-First Stability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove YeMind Zen's duplicate drag/view lifecycle logic and harden startup and persistence using simple-mind-map's native behavior.

**Architecture:** The editor delegates node movement, subtree movement, sibling insertion, layout recalculation, and viewport movement to simple-mind-map. YeMind Zen only persists native `data_change` and `view_data_change` outputs. Restored tabs wait for repository initialization, and repository writes are serialized so stale snapshots cannot overwrite newer ones.

**Tech Stack:** TypeScript, simple-mind-map 0.14.0-fix.3, SiYuan Plugin API, Vitest, Vite.
