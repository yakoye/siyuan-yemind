# YeMind v0.5.3 Native Clipboard Implementation Plan

> **Execution:** Follow upstream-first development and TDD. Do not add outer-frame, summary, relation-line, or cross-map features in this version.

**Goal:** Add reliable same-map node copy, cut, and paste by delegating tree filtering, subtree cloning, node removal, insertion, history, activation, and rendering to `simple-mind-map` native Render methods and commands.

**Architecture:** YeMind only exposes command and menu bindings. The map instance enables `disabledClipboard`, which is the native renderer's existing in-memory clipboard path, so clipboard permissions cannot break same-map operations. The actual data operations remain `Render.copy`, `Render.cut`, `Render.paste`, `CUT_NODE`, `PASTE_NODE`, and `INSERT_MULTI_CHILD_NODE`. Native `KeyCommand` continues to own Ctrl/Cmd+C/X/V, including its text-edit pause and canvas-focus checks.

**Upstream behaviors verified before implementation:**

- `Render.copyNode()` filters selected nodes through `getTopAncestorsFomNodeList`, preventing duplicate descendant subtrees.
- `copyNodeTree()` recursively clones the whole subtree and removes node UIDs and active state.
- `Render.cutNode()` filters out root nodes, clones selected top-level subtrees, removes them from parent data, clears activation, renders, and records history through the command system.
- `Render.pasteNode()` delegates to `INSERT_MULTI_CHILD_NODE`, which assigns fresh UIDs, activates pasted nodes according to native `createNewNodeBehavior`, expands the target, renders, and records history.
- Native system-clipboard reads may fail because of permissions. The same-map feature therefore uses the renderer's existing `beingCopyData` fallback instead of implementing a second clipboard data format.
