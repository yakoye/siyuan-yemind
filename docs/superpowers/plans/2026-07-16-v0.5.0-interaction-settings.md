# YeMind v0.5.0 Interaction and Settings Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix comment and todo interactions, add KMind-Zen-style general/shortcut settings, and deliver usable search plus map/outline/split views.

**Architecture:** Keep `simple-mind-map` as the editor engine. Put state transitions and view-model logic in pure TypeScript helpers with focused tests; keep SiYuan Dialog/Menu wiring in UI modules. Settings remain persisted through `SettingsStore`, while the custom settings dialog writes patches through that store so open editors update live.

**Tech Stack:** TypeScript, Vite, Vitest/JSDOM, SiYuan Plugin API, simple-mind-map Search plugin.
