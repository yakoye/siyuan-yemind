# Native Outer Frame Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Add stable single-node and continuous-sibling outer frames by exposing only simple-mind-map's native `OuterFrame` plugin lifecycle.

**Architecture:** `OuterFrame` remains the sole owner of range parsing, `groupId`, bounding-box rendering, text editing, history, delete shortcuts, and relayout behavior. YeMind Zen adds registration, normalized settings mapping, thin command methods, one context-menu entry, and a compact active-frame toolbar. No custom coordinates, range computation, or persistence sanitizer is introduced.

**Tech Stack:** TypeScript, Vitest, Vite, simple-mind-map 0.14.0-fix.3, SiYuan plugin API.
