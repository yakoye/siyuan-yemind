# YeMind v0.9.25 Import, Dark UI, and Outline Content Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Improve text-to-map import ergonomics, constrain imported node width without modifying source text, replace native theme/line selects with consistent dark-aware panels, and surface image/icon/clipart content in the structured outline with shared canvas actions.

**Architecture:** Keep parsing and map mutation pure in `outlineTreeImport.ts`; keep dialog rendering in `textToMapDialog.ts`; add a focused project-choice panel controller for theme/line menus; extend structured-outline blocks with presentation-only accessory data while preserving node text editing and tree reconstruction. Reuse existing `YeMindCommands`, marker picker, clipart picker, and image dialog so outline and canvas mutate the same node data.

**Tech Stack:** TypeScript, DOM APIs, SiYuan plugin UI, simple-mind-map, Vitest, Chromium smoke tests, Vite.
