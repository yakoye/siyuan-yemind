# Native Multi-Selection Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expose and verify simple-mind-map's native box selection, Ctrl/Cmd multi-selection, and batch subtree dragging without creating a second selection or drag implementation.

**Architecture:** Keep `Select`, `MindMapNode`, and `Drag` as the only owners of selection and batch movement. YeMind Zen adds only a persistent canvas-mode toggle, a small selected-node count presentation, accurate help/settings text, and contract tests that pin the upstream behavior we depend on.

**Tech Stack:** TypeScript, Vite, Vitest, simple-mind-map 0.14.0-fix.3, SiYuan plugin API.
