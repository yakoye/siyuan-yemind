# Checkpoints and Safe Restore Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add persistent named checkpoints with protected pre-restore snapshots and safe in-place editor reload.

**Architecture:** Store immutable map snapshots in a dedicated transactional `CheckpointRepository`. Coordinate checkpoint creation and restore through a small service that composes the existing `MapRepository`; keep `simple-mind-map` responsible only for rendering the restored data. Add one compact editor menu and one management dialog.

**Tech Stack:** TypeScript, Vitest, SiYuan Plugin API, simple-mind-map 0.14.0-fix.3.
