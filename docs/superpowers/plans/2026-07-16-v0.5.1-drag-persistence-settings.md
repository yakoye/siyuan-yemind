# YeMind v0.5.1 Drag, Persistence, and Settings Plan

**Goal:** Fix structured node dragging, viewport jumps, disappearing unnamed maps, and add the missing drag/layout/view settings required for stable daily use.

**Architecture:** Keep `simple-mind-map` as the layout and drag engine, but stop overriding its safe structured-drag defaults. Add pure normalization/config helpers so drag mode, spacing, and persisted view data are validated before reaching the editor. Make repository loading mutation-safe so startup cannot overwrite newly created maps.

**Tech Stack:** TypeScript, Vite, Vitest, SiYuan plugin API, simple-mind-map 0.14.0-fix.3.
