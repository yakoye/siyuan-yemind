# Generalization and Associative Line Completion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete YeMind Zen's native summary and associative-line workflow without replacing simple-mind-map's range-summary, line-editing, endpoint, control-point, history, or rendering logic.

**Architecture:** Keep `ADD_GENERALIZATION`, `REMOVE_GENERALIZATION`, `AssociativeLine.createLineFromActiveNode`, `showEditTextBox`, `removeLine`, and native plugin events as the sole behavior owners. YeMind Zen adds only dynamic menu labels, a lightweight line-state toolbar, four relevant settings, and a pure persistence sanitizer for stale relation targets after node deletion.

**Tech Stack:** TypeScript, Vitest, Vite, simple-mind-map 0.14.0-fix.3, SiYuan plugin API.
