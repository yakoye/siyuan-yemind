# YeMind v0.9.13 interaction polish implementation plan

> **Scope:** Patch release for the eight user-reported regressions in the v0.9.12 overlay package. The archive remains an overlay and intentionally excludes the large `assets/` tree.

## Task 1: Add permanent regression scenarios

- Add isolated v0.9.13 suites for marker geometry, image hover/preview, relation selection, toolbar/panel interaction, standalone About entry, default map naming, multi-selection context menus, and hidden-tab rich-text measurement.
- Update `tests/suite-manifest.json` and the matching 15 domain entry files.
- Run the tests before implementation and confirm that the new assertions fail against v0.9.12 behavior.

## Task 2: Repair marker and image interaction

- Render marker sprites through a viewport-sized SVG pattern so the sprite sheet cannot enlarge the node content bounding box.
- Restore hover-only image controls with delete, magnifier, and resize tools; remove the v0.9.11 pinned-click/double-click behavior.
- Make the image preview backdrop lighter while keeping blur and keyboard/close behavior.

## Task 3: Repair canvas selection and rendering

- Use a stable blue active relation color with a modest active width.
- Snapshot multi-selection during capture-phase right-click and restore it before opening the node context menu.
- Relocate simple-mind-map rich-text measurement elements outside hidden tab containers, re-render once after relocation, and remove the measurement elements on map destruction.

## Task 4: Polish panels and About placement

- Restore hover feedback for Structure and Style toolbar buttons.
- Close the Structure gallery on outside pointer interaction.
- Use separate compact dimensions for project style and node style panels and lay node controls out more densely.
- Remove About from Settings and add a standalone About dialog directly after Settings and before Diagnostics in the top-bar menu.

## Task 5: Update new-map defaults and release identity

- Keep the document/file title as `未命名导图`, set the center node to `中心主题`, and set both initial child nodes to `新节点`.
- Update version surfaces, release notes, coverage matrix, product boundaries, overlay notice, and verification documentation for v0.9.13.

## Task 6: Complete release verification

- Run structure validation, the full Vitest suite, TypeScript checking, production build, and built-entry syntax checking.
- Create `siyuan-yemind-v0.9.13-overlay.zip` without user data, `node_modules`, maps, settings, checkpoints, or the large `assets/` tree.
- Run ZIP CRC, fully extract the ZIP, run `npm ci`, and repeat tests, TypeScript checking, production build, and entry syntax checking from the extracted package.
