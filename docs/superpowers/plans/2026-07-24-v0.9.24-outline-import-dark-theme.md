# YeMind v0.9.24 Outline Import and Dark Theme Plan

## Goal

Add outline text-to-map import, align outline editing/context-menu behavior with the agreed specification, remove default leaf nodes, replace the rich-text π glyph with the shared formula icon, and make host light/dark switching visually and geometrically stable.

## TDD sequence

1. Add executable regression tests for text parsing/import, default-map shape, empty-row Enter promotion, formula icon markup, exact outline-menu labels, dark project controls, and appearance-transform restoration.
2. Run the new tests against v0.9.23 and confirm they fail for the intended missing behavior.
3. Implement the smallest pure parser/tree-transform and appearance-transaction changes.
4. Implement outline context menu, dialog, controller operations, Enter behavior, and formula icon.
5. Implement dark-mode styling/state propagation for Theme/Line controls and native lists.
6. Build the offline bundle and run all structure, syntax, offline, and Chromium regressions.
7. Update version/docs, package a flat ZIP without user data, extract it into a clean directory, and repeat verification.

## Acceptance criteria

- The outline context menu has the exact requested order and labels.
- Cutting the current line clears only its text and preserves its node and descendants.
- Unicode tree, Windows Tree, indentation, Markdown, numbered outline, and plain-line input parse with preview and mode-specific placeholders.
- Import is a single tree replacement and supports append-under-current or replace-current-title modes.
- A new map contains only the center topic.
- Enter on an empty outline row promotes it one level; at the root-child boundary it removes the empty row and focuses the root.
- The rich toolbar formula control contains the shared `iconMath` SVG and no π glyph.
- Appearance redraw restores view transform and does not persist internal theme-switch view changes.
- Zero-size hidden canvases defer full appearance redraw.
- Theme/Line controls and native option lists are visible in dark mode, including focus/open state.
- Final ZIP is flat, deterministic, CRC-valid, and excludes user data/build caches.
