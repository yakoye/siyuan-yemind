# YeMind v0.5.22 展开、编辑删除与右键平移设计

日期：2026-07-20

## 问题

1. 折叠节点的数字按钮和大纲三角收到了点击，但无法恢复展开，Root 同样受影响。
2. 分屏和纯大纲刚进入编辑或编辑过程中，Backspace/Delete 可能被全局快捷键提前解释为节点删除。
3. 画布右键需要支持平移；发生实际移动后，松开右键不能再弹出右键菜单，并应显示抓手光标。

## 根因

### 折叠节点无法展开

`simple-mind-map` 在节点折叠后可能不再保留可见的 `node.children` 实例，但完整子树仍存在于 `node.nodeData.children`。旧命令适配器使用 `node.children.length` 判断节点是否有子节点，因此把折叠分支误判为叶节点，在调用 `SET_NODE_EXPAND` 前直接返回。

### 编辑删除串入结构删除

大纲 Quill 激活后会异步把焦点转入 `.ql-editor`。在这个极短窗口内，键盘事件目标仍可能是外层 `[data-outline-editor]`。旧 `isEditableTarget()` 不认识该宿主，全局捕获阶段先执行节点删除。即使焦点已进入 Quill，事件继续冒泡到 `simple-mind-map` 的 window 快捷键也存在二次解释风险。

### 右键平移与菜单冲突

上游只在“选择优先”模式中使用右键平移；“平移优先”模式的右键仍只产生菜单。上游也没有把“发生拖动”状态暴露给 YeMind 的菜单入口。

## 设计

### 展开命令

- 子节点存在性优先读取 `node.nodeData.children`；兼容回退到 `getData('children')` 和 `node.children`。
- 数字按钮、大纲三角、Root 和普通分支统一调用 `setNodeExpandedByUid(uid, true)`。
- 大纲三角不再先执行导航命令，避免选择/聚焦副作用干扰展开事务。

### 大纲删除隔离

- `[data-outline-editor]` 及其全部后代始终视为可编辑目标。
- Root 捕获阶段若检测到活动大纲编辑器或其焦点后代，立即退出，不执行结构快捷键。
- 大纲冒泡阶段对 Backspace/Delete 只执行 `stopPropagation()`，不执行 `preventDefault()`：Quill/浏览器可以正常删除字符，但事件不会继续到上游 window 快捷键。
- 只有捕获阶段确认正文在按键发生前已经为空，才执行既有的“空节点结构删除”。

### 右键平移

引入独立手势状态机：

- 右键按下记录起点；5px 内仍视为普通右击。
- 超过阈值后进入拖动，编辑器增加 `is-canvas-right-dragging`，显示 `grabbing`。
- “选择优先”模式继续由上游移动视图，YeMind 只跟踪手势和抑制菜单。
- “平移优先”模式由 YeMind 按增量调用 `view.translateXY(dx, dy)`，因此左右两种模式均支持右键平移。
- 拖动期间或拖动结束后的下一次 contextmenu 被消费；未移动的右击正常打开菜单。
- 节点菜单和空白画布菜单共用同一抑制状态。

## 兼容性

- 不改变导图数据格式、历史格式和检查点格式。
- 不修改上游左键平移、框选和节点拖拽规则。
- 保留空白非 Root 大纲节点再次按 Backspace/Delete 时删除节点的既定语义。
