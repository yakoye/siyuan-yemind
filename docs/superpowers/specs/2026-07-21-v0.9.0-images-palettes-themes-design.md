# YeMind v0.9.0 image and color-system design

## Goals

1. Make node-image removal explicit and safe.
2. Provide an editor-local image preview that never loses the map context.
3. Keep note and comment hover previews unobstructed.
4. Expand rainbow lines into named reusable palettes.
5. Make whole-map themes use the same named palette language and matching backgrounds.

## Image controls

The node image overlay has three corners with distinct responsibilities:

- top-left: magnifier preview;
- top-right: trash-can deletion;
- bottom-right: proportional resize.

Deletion always asks for confirmation. Both the overlay control and the image dialog use the same wording. The lightbox is mounted inside the active YeMind editor, supports wheel zoom from 20% to 600%, 1:1 reset, outside-click, close-button and Escape dismissal.

## Hover badges

Note and comment badges keep accessible `aria-label` values but omit native `title` attributes. The custom hover preview is the only visible hover surface.

## Color schemes

Ten named schemes are shared by rainbow branches and whole-map themes:

- 晨曦
- 彩虹
- 活力
- 舞动
- 代码
- 和风
- 岛屿
- 玫瑰
- 薄荷
- 绿茶

Each scheme defines branch colors, background, text guidance and dark-background behavior where appropriate.

## Themes

The public theme list contains:

- YeMind 默认
- Ink Branch
- Material 3 Basic
- the ten named color schemes

Historical theme identifiers are accepted only by internal normalization and are never shown in the interface.

## Compatibility

v0.9.0 adds `rainbowScheme` to project style data. Missing or unknown values normalize to `null`, preserving existing maps. No node or repository schema version bump is required.
