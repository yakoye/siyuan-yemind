# YeMind v0.6.5 密度、搜索定位、颜色与富文本稳定性设计

## 目标

1. 重排整图密度语义：旧“紧凑”成为“默认”，旧主题默认成为“舒展”，新增更紧凑的“紧凑”，并支持自定义水平/垂直间距。
2. 整图样式面板支持点击外部关闭。
3. 思源全局搜索结果可稳定打开导图并定位到节点，即使目标标签页正在加载。
4. 节点样式中的填充、边框和文字颜色统一使用富文本同款色板、HEX/RGB 双向输入、重置默认和系统颜色选择器。
5. 修复画布节点双击编辑时文字变白/不可见、无法局部选择和无法弹出富文本工具栏的问题，并建立永久回归链。

## 方案比较

### 密度

- 仅改三个预设值：简单，但不能满足用户自定义。
- 用两个全局缩放比例：抽象，用户难理解。
- 三个预设加“水平/垂直”自定义：直观、兼容现有数据。采用。

### 全局搜索定位

- 点击后固定延时定位：时序脆弱，不采用。
- 搜索结果直接操作当前 DOM：无法处理尚未打开的导图，不采用。
- 标签句柄在编辑器未就绪时保存待定位 UID，编辑器挂载后消费：稳定且沿用现有标签注册结构。采用。

### 节点颜色

- 保留三个原生 color input：与富文本交互不一致，不采用。
- 为节点样式复制一套相似面板：易漂移，不采用。
- 抽取共享色板标记与颜色数据，节点样式使用同一交互模型：采用。

### 富文本可见性

上游 RichText 会把节点文本颜色写入编辑包装器，但宿主主题和项目 CSS 可能覆盖；实时渲染模式又使编辑器背景透明。进入编辑后由 YeMind 根据当前节点实际文字色和上游背景计算结果，显式同步编辑包装器和 Quill，保证文字与背景有可见对比，同时保留局部富文本颜色。

## 数据兼容

`ProjectStyle` 新增可选 `customMarginX/customMarginY`，`density` 增加 `custom`。旧数据继续规范化；原来的 `compact/default/comfortable` 在 v0.6.5 按新的产品语义解释，不修改存储版本。

## 验收

- 紧凑间距小于默认，默认等于 v0.6.4 紧凑，舒展等于主题原间距。
- 自定义水平/垂直间距保存、检查点恢复并实时生效。
- 样式面板内部操作不关闭，点击外部关闭。
- 搜索结果在已挂载和正在加载的标签页都能定位。
- 三类节点颜色都显示统一色板并支持 HEX/RGB。
- 浅色、深色、局部文字色节点双击后文字可见；拖选部分文字后工具栏出现并只格式化选区。
