# YeMind v0.5.23 画布富文本选区回归设计

日期：2026-07-20

## 问题

右键拖动画布后的结束事件只依赖 map-local `mouseup`。鼠标在画布外释放或宿主窗口失焦时，编辑器可能长期保留 `is-canvas-right-dragging`。旧 CSS 又把 `user-select:none` 施加到整个 `.ymz-editor`，因此随后双击节点虽然可能创建 Quill 编辑层，但用户无法自然拖选文字，也无法触发选区工具栏。

同时，快捷键隔离只识别 `.ql-editor` 和大纲宿主，没有把画布富文本包装器整体识别为编辑区域，存在 Delete/Backspace 被外层节点快捷键重新解释的风险。

## 设计

1. 右键拖动同时监听 map mouseup、window capture mouseup 和 window blur。
2. `before_show_text_edit` 主动取消残留右键拖动。
3. `user-select:none` 只作用于画布表面，不再作用于整个编辑器根节点。
4. 画布 Quill 包装器、容器和编辑区显式设置 `user-select:text` 和文本光标。
5. 富文本编辑层内部 pointer/mouse/dblclick/contextmenu 仅阻止传播，不阻止默认文字选择。
6. `.smm-richtext-node-edit-wrap` 整体归类为 editable target。
7. 用真实 SVG DOM 双击和真实 Quill range 验证工具栏及局部格式化，而不是只检查源码字符串。

## 长期回归范围

- Root/普通分支折叠数字和三角展开；
- 节点 `+ / − / 数字` 显示契约；
- 分屏、大纲和画布 Delete/Backspace 编辑隔离；
- 右键短按菜单与右键拖动抑制菜单；
- 节点双击、局部文字选择、工具栏显示及局部格式化。
