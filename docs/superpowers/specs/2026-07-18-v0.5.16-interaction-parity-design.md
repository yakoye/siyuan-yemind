# YeMind v0.5.16 交互对齐设计

日期：2026-07-18

## 目标

在 v0.5.15 稳定基线上完成八项已确认交互：

1. 移除激活节点右侧三点菜单按钮；
2. 批注节点入口只显示图标，不显示数量；
3. 颜色面板保留色板、重置默认、更多颜色，并显示 RGB 与 HEX；移除“吸管”入口；
4. 分屏和纯大纲中的 Backspace/Delete 始终由富文本编辑器删除文字，不再隐式删除节点；
5. 增加空白画布右键菜单；
6. 节点右键菜单保持完整长菜单，并使用紧凑、细窄滚动条；
7. 禅模式右上角退出胶囊默认收起，移入/聚焦后展开，点击退出；
8. 选中节点后支持粘贴图片、把本地图片拖入节点，并使用上游图片插件等比例缩放。

## 官方源码研究结论

用户提供的 KMind Zen 0.33.0 生产包明确区分画布菜单和节点菜单。节点图片命令包含添加、复制、粘贴、删除、设置位置和设置尺寸；图片模型保存宽、高与纵横比，并在调整时保持比例。官方 README 也明确说明禅模式右上角显示当前状态并可退出、节点图片支持调整。

当前 `simple-mind-map` 已提供画布 `contextmenu`、节点 `node_contextmenu`、根节点居中、整图适配、视图重置、展开/折叠全部节点、`SET_NODE_IMAGE` 和 `NodeImgAdjust`。因此本版只补 YeMind 适配层，不复制官方私有文档模型，也不实现第二套图片几何或历史。

## 方案比较

### 方案 A：复制官方完整菜单和图片内核

优点：外观和行为最接近官方。

缺点：官方生产包依赖私有文档模型、命令总线和持久化结构，会形成第二套状态与历史，破坏当前仓库边界。

不采用。

### 方案 B：仅用 CSS 模仿截图

优点：改动小。

缺点：无法补齐画布菜单、图片粘贴/拖入和删除语义，仍会出现“看起来像但行为不一致”。

不采用。

### 方案 C：适配官方语义到现有命令层（采用）

- 菜单继续使用思源原生 `Menu`，增加 YeMind 专属 class 和分组；
- 图片继续写入上游 `SET_NODE_IMAGE`，尺寸调整继续由已注册的 `NodeImgAdjust` 完成；
- 粘贴/拖入只识别 `image/*`，读取第一张有效图片；
- 大纲删除键不再映射结构命令；节点删除只保留在明确的菜单或现有结构命令路径；
- 不改变导图 JSON、检查点或设置文件版本。

## 组件设计

### 节点装饰

`createNodePostfixContent()` 只负责批注图标。删除三点按钮生成器、事件、宽度和设置开关。旧设置文件即使仍带 `showNodeMenuButton`，`SettingsStore` 也会忽略未知字段。

批注入口的 title/aria-label 固定为“批注”，不把数量暴露到节点表面；批注弹窗内仍保留全部批注。

### 颜色面板

新增纯函数把 `#RGB`、`#RRGGBB`、`rgb()` 和 `rgba()` 规范化为大写 HEX 与 `RGB(r, g, b)`。面板结构为：

- 52 个色块；
- 当前颜色示意；
- HEX 只读值；
- RGB 只读值；
- 重置默认；
- 更多颜色。

“更多颜色”继续调用原生 `<input type=color>`。默认/继承状态显示“默认/继承节点颜色”，不写入白色或透明色。

### 大纲删除语义

`resolveOutlineKeyAction()` 对 Backspace/Delete 返回 `none`，让活动 Quill 自己处理字符、选区和段落。即使当前行为空，也不删除节点。结构删除通过节点右键菜单等明确动作完成。

### 右键菜单

`openCanvasContextMenu()` 订阅上游画布 `contextmenu`，只提供整图和视图动作：

- 定位到中心主题；
- 适配全部节点；
- 重置缩放与位置；
- 整理布局；
- 展开/折叠全部节点；
- 进入/退出禅模式；
- 进入/退出只读模式。

节点菜单继续覆盖已有完整节点动作，不削减能力。节点菜单元素增加 `ymz-context-menu--node`，限制最大高度并使用窄滚动条；画布菜单使用 `ymz-context-menu--canvas`。

### 禅模式胶囊

DOM 保持一个可访问按钮，内含图标和文字。默认只显示图标；hover 或 `focus-visible` 时展开文字。点击和 Escape 都走已有 `toggleZen(false)`。

### 图片粘贴、拖入和缩放

新增 `nodeImageInput` 适配层：

- 从 `ClipboardEvent.clipboardData.items/files` 查找第一张 `image/*`；
- 从 `DragEvent.dataTransfer.items/files` 查找第一张 `image/*`；
- 将客户区坐标转换为导图坐标，遍历普通子节点与 `_generalizationList[].generalizationNode` 命中目标；
- 使用现有 `loadImageFileSelection()` 读取 Data URL 和自然尺寸；
- 粘贴时使用当前主节点；拖入时激活命中节点；
- 只读、无有效目标或非图片时不吞掉原事件；
- 写入 `{url,title,width,height,custom:false}`。

等比例缩放不重复实现，由已注册的上游 `NodeImgAdjust` 完成。

## 错误处理

图片读取或测量失败显示思源错误消息，不留下未处理 Promise。只读、无节点或非图片数据不执行写入。原生系统颜色选择取消不报错。菜单回调继续写入现有诊断记录。

## 测试策略

严格 RED → GREEN：

- 节点装饰：三点按钮不存在，批注数量元素不存在；
- 颜色：无 eyedropper，RGB/HEX 随当前格式更新；
- 大纲：空行 Backspace/Delete 不返回 remove；
- 菜单：画布菜单动作与节点菜单 class/滚动契约；
- 禅模式：DOM 和 CSS 收起/展开契约；
- 图片：剪贴板/拖入文件识别、目标判断、自然尺寸写入、失败处理；
- 集成：编辑器订阅画布 contextmenu、paste、dragover、drop，继续注册 `NodeImgAdjust`。

最终运行全量测试、TypeScript、生产构建、入口语法、依赖审计、ZIP 完整性和解压后入口语法。

## 不在本版范围

不实现官方图片上下左右位置系统，不引入官方文档卡/块卡，不增加图片压缩配置，不改变思源资产存储方式，不重构现有导图仓库或历史系统。
