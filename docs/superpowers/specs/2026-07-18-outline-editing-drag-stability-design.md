# YeMind Zen v0.5.18 大纲编辑、拖拽与模糊显示重构设计

日期：2026-07-18  
基线：YeMind Zen v0.5.17  
官方参考：KMind Zen 0.34.0（用户提供的 `siyuan-kmind-zen-v0.3.4.zip`，以包内版本为准）

## 1. 目标

本版集中解决分屏和纯大纲模式的编辑与拖拽稳定性，并同步修正折叠、公式图标和模糊显示：

1. Backspace/Delete 在编辑状态下连续工作，不因一次文字变化退出编辑。
2. 仅在明确切换节点、点击编辑区外部、按 Escape 或销毁视图时结束编辑；节点文字为空本身不删除节点。
3. 移除六点拖拽把手，使用整行按住并拖动；文本编辑和文字选择不能被误判为拖拽。
4. 拖拽同时利用纵向落点和横向缩进，支持同级排序、成为目标子节点以及向左降低层级。
5. 拖动不得把节点文字提交为空，不得产生画布上的空白小圆圈节点。
6. 折叠三角必须可反复折叠和展开。
7. 公式入口使用更接近数学排版的粗体斜体 π 图标。
8. “模糊”使用高斯模糊/毛玻璃视觉，不再显示为纯色块；悬停恢复内容。

## 2. 官方实现研究结论

KMind Zen 0.34.0 的大纲不是若干独立 textarea，而是一个连续的 ProseMirror/Tiptap 文档：每一行是 `outlineItem` 块，编辑事务、selection、IME 和拖拽都发生在同一编辑器中。它还使用稳定实体键、独立折叠属性和几何化 drop slot。

完整照搬不可行，原因是官方编辑器依赖其私有文档模型、命令总线、资产系统、撤销历史和 ProseMirror 扩展；直接复制会在 YeMind 内形成第二套导图结构和历史。

思源自带大纲同样依赖思源块模型、Protyle 编辑器和内核事务，不能安全地直接嵌入 `simple-mind-map` 节点数据。但可移植其交互原则：稳定 DOM、编辑期间不重建、整行候选拖拽、横向层级意图、明确 drop slot、外部焦点才提交。

## 3. 方案比较

### 方案 A：完整移植官方连续 ProseMirror 大纲

优点：最接近官方行为。  
缺点：必须引入官方私有 schema、命令、文档同步和历史；与 YeMind 的 `simple-mind-map` 冲突，体积和维护风险最高。  
结论：不采用。

### 方案 B：直接复用思源 Protyle/大纲 DOM

优点：视觉和输入体验接近思源。  
缺点：思源大纲面向块 ID，不面向导图 UID；内部 API 不是插件稳定契约，拖拽结果也无法天然映射到导图树。  
结论：不采用。

### 方案 C：移植官方事务和拖拽机制，保留 YeMind 数据内核

做法：保留单活动 Quill 和 `simple-mind-map` 命令，把 DOM patch、编辑生命周期和拖拽意图重构为官方式稳定机制。  
优点：解决当前根因，保留既有富文本、历史和持久化；变更边界清晰。  
缺点：需要自己实现 pointer drag 和 keyed reconciliation。  
结论：采用。

## 4. 已确认根因

### 4.1 输入一次后退出编辑

v0.5.17 的 `patchOutlineTree()` 虽保留活动行对象，但把所有行加入 `DocumentFragment` 后再次 `appendChild()`。这会把仍在原父容器中的活动行移出并重新插入，浏览器会丢失 contenteditable/Quill 的真实焦点和 selection。

文字变化 160ms 后自动提交，`SET_NODE_TEXT` 同步触发 `data_change`，随后 patch 移动活动行，因此表现为 Backspace 不生效或 Delete 删除一个字符后退出。

### 4.2 拖动后出现空白小圆圈

拖拽前后的 `flush → data_change → renderOutline → flush/detach` 存在同步重入；再叠加活动 Quill DOM 被移动，可能从失焦或已被重排的编辑器读取空 HTML，并把空值写回节点。画布随后渲染出没有文字的极小节点。

### 4.3 折叠后不能展开

折叠操作同时使用 `pendingOutlineFocus` 驱动 detach、patch 和异步 focus 恢复。焦点恢复与下一次按钮点击可能竞争，且通用 click handler 会在按钮动作后继续受行选择行为影响。折叠应独立于编辑焦点：先提交当前草稿，再切换展开状态，只有原本正在编辑同一行时才恢复编辑。

## 5. 编辑生命周期

新增明确状态：

- `idle`：无活动编辑器。
- `editing`：活动 Quill 保持挂载。
- `composing`：IME 组合输入；禁止结构命令和自动提交。
- `committing`：把草稿写回一次，禁止同步重入。
- `dragging`：编辑器先安全提交并 detach，随后只执行结构命令。

退出编辑的允许原因：

- 用户点击另一行并开始编辑；
- 指针点击大纲和富文本浮层之外；
- Escape 取消；
- 视图销毁或切换文档；
- 开始结构拖拽或结构命令。

以下行为不得退出编辑：普通输入、Backspace、Delete、自动保存、画布选择同步、颜色/公式等编辑器局部浮层交互。

## 6. 稳定 DOM patch

`patchOutlineTree()` 改为 keyed in-place reconciliation：

1. 内容和可见结构签名未变化时，不移动任何现有行，只更新非活动行内容和属性。
2. 结构签名变化时，由 `YeMindEditor` 先安全提交并 detach，再执行有序 DOM patch。
3. 行顺序只在实际顺序变化时使用 `insertBefore()`，不再把全体行放入 fragment 后重新 append。
4. 活动 UID 的内容在编辑期间完全由 Quill 拥有，模型回流不得覆盖。

## 7. 整行 pointer 拖拽

移除六点按钮。每一行作为拖拽候选，但保留文字编辑能力：

- 非编辑区按下并移动超过 6px：立即开始拖拽。
- 编辑正文内按下：允许光标和选区；只有持续按住 260ms 且随后移动超过 6px 才进入拖拽。
- toggle、链接、复选框、按钮等交互控件不启动拖拽。
- 使用 Pointer Events、窗口级 pointer 追踪和自绘 ghost/drop indicator，不依赖 HTML5 `draggable`。
- Escape、pointercancel、丢失 capture 均安全取消。

### 7.1 落点语义

纵向：目标行上 25% 为 before，下 25% 为 after，中间为 inside。  
横向：以目标正文左边界和 22px 层级宽度计算期望深度：

- 向右至少一个层级：inside，成为目标子节点；
- 目标深度附近：before/after，同级排序；
- 向左一个或多个层级：把目标解析到相应祖先层级，再 before/after，实现降低层级。

最终变更仍只调用上游 `MOVE_NODE_TO`、`INSERT_BEFORE`、`INSERT_AFTER`，不直接改写树。

## 8. 折叠与展开

- toggle 使用独立 pointer/click 路径并阻止行拖拽和行点击穿透。
- 展开状态以最新模型为准，不依赖旧 DOM dataset。
- 操作前仅在需要时提交当前活动编辑器。
- 折叠同一活动行时，操作完成后恢复到同一行；展开后也可继续编辑。
- 无子节点时显示对齐占位，不显示可点击圆点。

## 9. 公式图标

工具栏仍保持可访问文本“插入公式”，视觉内容改为专用 `<span>`：

- 字体优先 `STIX Two Math`、`Cambria Math`、`Times New Roman`；
- 粗体、斜体、略微横向拉伸；
- 视觉尺寸约 22px；
- 不把图标字符写入节点。

## 10. 模糊效果

继续保留已有 `background + transparent color` 数据以兼容旧文件，但显示层覆盖为：

- 恢复可见文字色后施加 `filter: blur(4px)`；
- 使用低透明度、可透出背景的圆角底层与 `backdrop-filter`；
- 禁止选择隐藏内容；
- 悬停时平滑取消 blur、恢复不透明度和可选择状态；
- 画布、分屏和纯大纲使用同一组规则；
- “完全隐藏”设置继续保持真正不可见。

## 11. 测试范围

### 编辑

- 连续 Backspace/Delete 不触发 detach。
- 自动提交后的 patch 不移动活动行，`document.activeElement` 仍为 Quill root。
- 清空文字只写回空文本，不删除节点。
- IME 期间不提交、不执行结构命令。
- 模型仅更新文字时结构签名不变；结构变化时明确 detach/restore。

### 拖拽

- HTML 不再包含六点把手或 `draggable=true`。
- 普通正文点击/短距离移动不启动拖拽。
- 长按正文和直接拖动行空白区可以启动。
- before/inside/after 与向左降级层级正确解析。
- 不能拖到自身或自己的后代。
- 拖动前提交一次，拖动后节点正文保持不变。
- 取消拖动不执行结构命令。

### 折叠、图标、模糊

- 折叠/展开可连续往返。
- 无子节点的占位不可点击。
- 公式按钮包含数学排版 π，不再是普通文本按钮。
- blur 模式含 `filter/backdrop-filter`，并同时覆盖画布和大纲。

## 12. 非目标

- 不引入第二套 ProseMirror 文档或撤销历史。
- 不复制官方私有资产、协作和文档存储模块。
- 不改变导图文件格式。
- 不在本版重做备注、批注、颜色面板等已交付功能。
