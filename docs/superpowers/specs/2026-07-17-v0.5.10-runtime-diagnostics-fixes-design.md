# YeMind Zen v0.5.10 真实运行问题修复设计

## 目标

基于用户上传的两份思源 v3.7.2 诊断包，修复真实运行中已确认的问题，不增加新的导图业务功能。

## 诊断结论

1. 三张真实导图均存在于 `maps.json`，日志中没有真实 `map-deleted` 事件。
2. 诊断自检当前直接在正式 `MapRepository` 中创建临时导图，再在 `finally` 中删除，因此 Dock 会短暂多出一张导图并随后消失，造成“文件被删除”的错觉，也会触发正式仓库订阅与活动导图变化。
3. 日志捕获到 `容器元素el的宽高不能为0`。触发点是切换到纯大纲后仍调用 `mindMap.resize()`。
4. 第三张真实导图多次收到打开请求，但没有对应编辑器挂载事件，需处理隐藏/零尺寸容器和陈旧标签句柄。
5. 待办复选框当前复用了右键三态逻辑，导致已完成后再次点击直接删除待办。
6. 上游 Drag 已计算当前目标父节点和插入位置，但现有提示线不够显眼；只增强上游目标提示，不改变结构化拖拽规则。

## 修复范围

### 1. 隔离诊断生命周期测试

- 自检使用独立临时存储和独立 Map/Checkpoint Repository。
- 不向正式导图列表插入临时导图。
- 不修改正式活动导图。
- 临时存储始终在 `finally` 中清理。
- 同一时间只允许一个自检任务。

### 2. 启动、Dock 与标签恢复

- Dock 在仓库加载完成前显示“正在加载”，加载后强制重新渲染。
- 标签编辑器只在容器宽高大于 0 后创建。
- 纯大纲模式不调用导图库 `resize()`。
- 从大纲回到导图/分屏时，等待容器可见后安全 resize。
- 标签注册增加存活检查，陈旧句柄不能阻止重新打开导图。

### 3. 待办交互

- 复选框：未完成 ↔ 已完成，永不删除。
- 右键菜单：添加待办 → 待办完成 → 删除待办。
- 删除待办只能通过右键“删除待办”。

### 4. 拖拽目标虚线

- 继承上游 Drag 插件，只增加目标引导线样式。
- 使用上游已经计算的 `overlapNode / prevNode / nextNode`。
- 虚线连接当前拖拽克隆与上游判定的目标父节点。
- 不改变 `MOVE_NODE_TO / INSERT_BEFORE / INSERT_AFTER`。
- 无有效目标时隐藏引导线。

## 非目标

- 不增加自由拖拽。
- 不重写节点重排算法。
- 不改变导图数据格式。
- 不增加新的导图功能。

## 验收

1. 运行诊断自检时，正式导图列表数量不变化。
2. 三张真实导图重启后都能显示并打开。
3. 隐藏标签或纯大纲模式不再产生零尺寸容器错误。
4. 待办复选框可反复勾选和取消勾选，待办仍存在。
5. 右键菜单在已完成状态显示“删除待办”。
6. 拖动节点时出现清晰虚线，指向当前上游目标父节点。
7. 全部既有测试、类型检查、生产构建和 ZIP 复检通过。
