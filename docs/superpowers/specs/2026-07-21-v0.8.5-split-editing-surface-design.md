# YeMind v0.8.5 分屏编辑表面所有权设计

## 问题

分屏同时存在画布富文本编辑器与大纲 Quill 编辑器。旧实现把“结构变化时仍有活动大纲编辑器”解释为“重绘后必须恢复该行焦点”。当用户已经切换到画布并用 Tab/Enter 创建新节点时，这个推断会复活之前编辑的大纲节点。

## 根因

旧流程：

1. 大纲节点 A 留有活动 `OutlineRichTextController`；
2. 用户点击画布节点 C；
3. Tab/Enter 创建 D 并触发 `data_change`；
4. `renderOutline()` 发现结构变化和活动 A；
5. 自动生成 A 的 `pendingOutlineFocus`；
6. 重绘后再次激活 A。

问题不在新建命令，而在编辑焦点没有明确所有者，且焦点恢复是由模型变化隐式推断的。

## 设计

新增 `EditingSurfaceCoordinator`：

- 所有者只有 `none / canvas / outline`；
- 只有大纲自身命令可以显式创建焦点恢复票据；
- 画布接管时清空票据并递增代次；
- 异步恢复必须校验票据仍是当前代次；
- 外部结构变化只提交并关闭旧大纲编辑器，不创建恢复票据。

## 画布接管入口

- 画布捕获阶段 `pointerdown`；
- `node_active` 且文档焦点不在活动大纲编辑器；
- `before_show_text_edit`；
- 从分屏/大纲切换到纯导图。

## 保留行为

大纲内以下命令仍可显式恢复焦点：

- Enter 插入同级；
- Root Enter / 指定操作插入子级；
- Tab / Shift+Tab 缩进与减少缩进；
- 空节点删除后的相邻节点；
- 折叠与展开后的当前节点。

普通文本提交不改变结构签名，因此活动 Quill 不会被重建。
