# YeMind Zen v0.6.8 原生全局搜索融合设计

日期：2026-07-21

## 目标

将 YeMind 搜索结果真正融合进思源原生全局搜索的三段式结构：

1. 顶部搜索条件区保持完全原生；
2. YeMind 节点作为 `#searchList` 中的标准结果项；
3. 选中 YeMind 结果时在 `#searchPreview` 中显示大纲上下文；
4. 支持鼠标、方向键、Enter、双击、Alt+Enter 与 Alt+单击；
5. 思源重绘结果列表后自动恢复，且不重复插入。

## 当前问题

v0.6.7 将结果面板插入 `searchInput` 的父容器，因此：

- 结果挤占搜索栏；
- 不能使用原生结果选中状态；
- 无法使用原生预览区域；
- 搜索框失焦或宿主重绘时容易丢失；
- 键盘导航和鼠标操作形成两套互不相通的逻辑。

## 方案

### 搜索结果

YeMind 结果使用思源结果条目的视觉结构：

- `b3-list-item` 外观；
- 左侧 YeMind 图标；
- 主文本为匹配节点内容；
- 右侧 `b3-list-item__meta` 显示导图标题和节点路径；
- 搜索词用 `<mark>` 高亮；
- 备注、批注、标签、链接和待办显示轻量来源标记。

结果分组标题只用于说明，不进入键盘选中序列。

### 预览

不创建完整可编辑导图实例。选中结果时，在 `#searchPreview` 上方显示只读大纲覆盖层：

- 显示导图标题和完整节点路径；
- 自动展开目标节点祖先；
- 显示祖先的同级节点；
- 显示目标节点一层子节点；
- 当前匹配节点使用主题绿色高亮；
- 大型导图最多渲染 200 行。

覆盖层不销毁思源原生 Protyle；切回普通思源结果时直接隐藏并恢复原生预览。

### 交互

- 单击：选中 YeMind 结果并切换大纲预览；
- 双击 / Enter：打开导图并定位节点；
- Alt+单击 / Alt+Enter：在右侧分屏打开；
- 上下键：在 YeMind 结果中移动，并可与第一个原生结果衔接；
- PageUp / PageDown：在 YeMind 分组首尾快速移动；
- Esc：交给思源关闭全局搜索；
- 打开具体节点失败时仍保证打开导图。

### 宿主重绘

为搜索窗口根节点安装轻量 MutationObserver：

- 仅在已存在查询和匹配结果时工作；
- 检测 `#searchList` 被清空或替换后重新挂载；
- 检测 `#searchPreview` 被重建后恢复当前预览；
- 使用微任务合并，避免观察自身 DOM 修改形成循环；
- 搜索输入为空、无匹配或元素断开时清理结果和预览。

## 数据范围

继续搜索：

- 导图标题；
- 节点正文；
- 备注；
- 批注；
- 标签；
- 链接；
- 待办。

匹配排序：节点正文精确匹配、前缀匹配、包含匹配、标题、备注/批注、其他元数据。

## 发布门禁

- 真实思源全局搜索 HTML fixture 中，搜索栏内部不得出现 YeMind 结果；
- YeMind 条目必须位于 `#searchList`；
- 选中结果必须在 `#searchPreview` 显示大纲；
- 普通思源结果点击后恢复原生预览；
- 单击不打开，双击与 Enter 打开；
- Alt 操作传递右侧打开语义；
- 宿主重绘后结果不丢失、不重复；
- 全量旧回归、TypeScript、生产构建和发布 ZIP 二次验证通过。
