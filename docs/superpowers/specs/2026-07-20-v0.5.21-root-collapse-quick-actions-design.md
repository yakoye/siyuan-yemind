# YeMind v0.5.21 Root 折叠与快捷控制设计

## 根因

上一版存在三个互相叠加的限制：

1. 编辑器启动发现 Root `expand=false` 时会强制改写成 `true`；
2. 大纲展平无条件把深度 0 当作展开；
3. Root 行不生成折叠按钮。

因此 Root 折叠会在数据同步或重新打开时失效。普通节点数字入口虽然存在，但与始终显示的 `+` 混合，且缺少点击路由的直接回归。

## 方案

- Root 和普通分支统一使用节点原生 `data.expand`；
- 大纲 Root 行与普通分支使用同一个 disclosure 按钮；
- 折叠节点只显示数字，数字点击调用显式 `setNodeExpandedByUid(uid, true)`；
- 未折叠节点只有在被选中时才显示快捷控制；
- `+` 与 `−` 使用原紧凑 `−` 的 15×15 px 规格；
- 不建立第二套折叠集合，不改变导图存储格式或上游历史。
