# YeMind v0.8.0 品牌与工程重命名设计

日期：2026-07-21

## 目标

1. 使用用户提供的新图标，颜色统一为 `#176B50`，生成 32、64、128、512 像素透明 PNG。
2. 所有当前运行界面、设置、搜索、诊断和发布文档的公开名称由 `YeMind` 改为 `YeMind`。
3. npm 工程名、源码插件类、源码文件和发布压缩包由 `siyuan-yemind-zen` / `YeMindZenPlugin` 改为 `siyuan-yemind` / `YeMindPlugin`。
4. 保留思源技术插件 ID `siyuan-yemind-zen`，确保旧安装升级后仍读取同一份 maps、settings 和 checkpoints 数据。

## 方案比较

### 方案 A：同时修改 plugin.json.name

优点：所有名称完全一致。

缺点：思源会把它识别为另一个插件，旧插件数据目录、标签恢复、插件 URL 和用户安装路径都会断开，需要高风险的跨插件数据迁移。

结论：不采用。

### 方案 B：只改显示名

优点：风险最低。

缺点：package、源码类、发布包仍残留 Zen，不满足“工程也改名”。

结论：不采用。

### 方案 C：公开品牌与工程重命名，技术 ID 保持兼容

- `displayName`、产品文案：`YeMind`
- npm/project/release archive：`siyuan-yemind`
- 入口类：`YeMindPlugin`
- 技术插件 ID：继续为 `siyuan-yemind-zen`

结论：采用。

## 图标处理

- 从用户提供的 RGB 图片中按绿色色差提取透明度，移除棋盘格背景。
- 所有非透明像素使用精确颜色 `#176B50`，边缘通过 alpha 保留抗锯齿。
- 内容按真实图形边界裁剪，居中放入正方形透明画布并保留安全边距。
- 输出：
  - `assets/yemind-icon-32.png`
  - `assets/yemind-icon-64.png`
  - `assets/yemind-icon-128.png`
  - `icon.png`（512×512）
- 思源 SVG 图标符号引用新的 32px 资源，不再绘制旧的 `Ye` 方块。

## 兼容边界

- `plugin.json.name` 和插件 URL 协议暂不修改。
- 存储文件名、TAB_TYPE、DOCK_TYPE 不修改。
- 历史版本文档允许保留旧版本当时的产品名称；当前入口文档和运行时代码必须改名。
