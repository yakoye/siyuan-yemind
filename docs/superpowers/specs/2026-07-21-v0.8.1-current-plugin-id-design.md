# YeMind v0.8.1 当前插件 ID 设计

## 目标

将当前思源插件 ID、安装目录、资源 URL、诊断身份和新协议链接统一为 `siyuan-yemind`，同时只在协议解析处接受历史 `siyuan-yemind-zen` 链接。

## 约束

- 用户已整体重命名安装文件夹，但思源 `loadData/saveData` 使用 `data/storage/petal/<插件ID>/`，因此正式数据需要从旧 ID 存储非破坏性复制。
- 发布包不得包含或覆盖 maps/settings/checkpoints 用户数据。
- ZIP 内不额外嵌套目录，以便直接解压覆盖。
- 不自动删除旧安装目录或旧 petal 存储。
- 仅当新 ID 对应文件缺失时复制，已有新数据绝不覆盖。

## 兼容

- 当前链接生成：`siyuan://plugins/siyuan-yemind?map=...`。
- 历史链接解析：同时接受 `siyuan-yemind-zen`。
- 旧恢复标签可能包含旧插件身份，升级前建议关闭，重启后重新打开。
