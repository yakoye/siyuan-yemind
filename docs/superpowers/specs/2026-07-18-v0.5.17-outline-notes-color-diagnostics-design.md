# YeMind Zen v0.5.17 设计：官方式大纲、备注悬停、颜色输入与诊断增强

日期：2026-07-18

## 目标

在 v0.5.16 基线上完成以下用户已确认需求：

1. 禅模式退出胶囊默认显示“● 禅”，悬停/键盘聚焦显示“● 退出禅模式”。
2. 分屏与大纲模式参考官方 KMind Zen 0.34.0 的持续编辑、行级拖拽与结构事务机制，并补齐中文输入法、焦点、删除、选区和撤销回归。
3. 每一条普通大纲节点支持拖拽：同级排序、成为子节点、移出层级；分屏与纯大纲行为一致。
4. 右键待办语义仅为“添加待办 / 删除待办”；完成状态只由节点复选框切换。
5. 颜色面板提供可编辑 HEX 与 RGB，合法输入实时双向转换和应用；颜色输入区域完全隔离节点/大纲快捷键。
6. 增加长文备注；备注和批注节点图标悬停自动预览；右键批注使用可靠的内置 SVG 图标。
7. 富文本“挖空/取消挖空”改为“模糊/取消模糊”，底层 cloze 数据保持兼容；公式入口 `Fx` 改为 `π`。
8. 增强诊断：编辑事务、IME、焦点、节点数变化、问题时间标记、版本一致性检查和事件去噪。

## 官方源码结论

用户提供的官方包内部版本为 0.34.0。官方更新记录明确包含：节点正文与大纲行图片粘贴和缩放、备注图片粘贴和尺寸调整、根节点 Enter 创建子节点，以及 0.33.0 的非法内容初始化和展开卡死修复。

官方大纲不是逐行 textarea，而是持续文档编辑器模型：

- 每个行实体带稳定 `entityId/rowKey/depth/orderPath`；
- 每行有独立 chrome、拖拽手柄和 drop slot；
- 文本编辑和结构拖拽属于不同事务；
- 结构操作前执行 flush，并对 missing/stale/busy 给出明确结果；
- 当前编辑器不会因为普通文字写回而被整行重建。

YeMind 不直接复制官方私有内核，而适配这些机制到现有 `simple-mind-map` 命令层。

## 方案比较

### 方案 A：继续修补当前整表重渲染

优点：修改量小。缺点：每次 `data_change` 都可能销毁活动 Quill；拖拽、IME 和选区会持续出现竞态。放弃。

### 方案 B：引入完整 ProseMirror/Tiptap 大纲内核

优点：最接近官方。缺点：会引入第二套编辑文档、历史和结构模型，与现有 `simple-mind-map` 冲突，兼容和体积风险过高。放弃。

### 方案 C：稳定行 DOM + 单活动 Quill + 显式结构事务（采用）

- 大纲行以 UID 为 key 增量 patch，不在普通文本提交时重建活动行；
- 单活动 Quill 保持 selection、IME 和草稿；
- 文本提交带 session/revision，忽略自身回流；
- 结构命令前 flush，失败则中止；
- 行拖拽只计算意图，最终调用上游 `MOVE_NODE_TO/INSERT_BEFORE/INSERT_AFTER` 或现有命令适配；
- 拖拽从专用手柄开始，避免与文字选择冲突。

该方案保留唯一导图结构、唯一历史和唯一持久化来源。

## 大纲编辑状态

状态：`idle -> focused -> composing/editing -> pending-commit -> committed`。

约束：

- `compositionstart` 到 `compositionend` 不执行结构快捷键和自动提交；
- 普通输入只更新活动 Quill，本地 160ms 去抖提交；
- 自身文本提交产生的 `data_change` 只 patch 非活动行；
- 外部结构变化仍保留活动 UID 和 selection，目标不存在时明确 detach；
- Backspace/Delete 在编辑器内部永不直接删除节点；节点删除只从非编辑状态的结构命令触发；
- Enter：根节点创建子节点，普通节点创建同级；Shift+Enter 为换行；Tab/Shift+Tab 调整层级；
- 上下方向键只在光标处于首/尾且没有选区时跨行，避免正常文字导航被劫持。

## 大纲拖拽

每行新增专用拖拽手柄。拖动时根据指针相对目标行分为：

- 上 25%：插入目标前；
- 中 50%：成为目标最后一个子节点；
- 下 25%：插入目标后。

限制：根节点不能作为普通节点移动；不能拖入自身或后代；只读模式禁用；拖拽前 flush 当前编辑器。拖拽结果通过命令适配层执行，并支持现有 undo/redo。

## 颜色输入

颜色面板结构：52 色板、重置默认、更多颜色、HEX 输入、RGB 输入。

- HEX 接受 `#RGB/#RRGGBB`；RGB 接受 `r,g,b` 和 `rgb(r,g,b)`；
- 合法值实时转换、同步原生颜色输入并应用；
- 输入不完整或非法时不应用，显示 `aria-invalid`；
- 面板打开时保留富文本 selection；输入焦点不触发节点快捷键；
- `Enter` 应用当前合法值并保持面板，`Escape` 回退到打开面板前的颜色并关闭；
- 色块点击应用后关闭；手动输入不自动关闭，便于微调。

## 备注与批注

### 备注

每节点一份长文备注，数据字段：

```ts
yemindNote?: {
  html: string;
  createdAt: number;
  updatedAt: number;
  width?: number;
  height?: number;
}
```

备注编辑器支持多段文本；本版实现安全富文本 HTML 和图片粘贴为 data URL，弹窗尺寸可拖拽并保存。节点存在备注时显示备注图标。

### 悬停预览

备注和批注图标 `pointerenter` 后 250ms 打开局部预览，`pointerleave` 延迟 180ms 关闭；鼠标进入预览则取消关闭。预览限制宽高并滚动，不显示批注数量。

## 诊断

新增事件：

- `outline-edit-start/focus/beforeinput/text-change/composition-start/composition-end/selection-change/commit/cancel/blur/destroy`
- `outline-structure-command/drag-start/drag-intent/drag-commit/drag-cancel`
- `outline-node-count-change/focus-owner-change`

新增“标记问题发生”，保存时间戳；导出时增加该时间点前 60 秒、后 20 秒摘要。`view-change` 进行短窗口聚合，避免淹没关键事件。

版本一致性比较 plugin manifest、package/build 常量和诊断服务版本；不一致在自检中失败。

## 兼容性边界

- 不改变现有 map storage schema 版本；新字段是节点可选字段。
- `note` 旧字符串如存在，读取时迁移为 `yemindNote.html`，但不破坏原字段。
- cloze 底层格式不变，只更换界面文字。
- 不引入官方私有内核、第二历史、第二导图仓库或永久每行 Quill。

## 验收

自动化覆盖颜色转换与隔离、待办菜单、禅胶囊、备注数据和悬停、批注图标、大纲输入/IME/删除/焦点、增量 patch、拖拽意图与结构命令、诊断标记和版本检查。
