# YeMind Zen v0.4.0 行内链接、代码块与设置设计

## 目标

在 v0.3.0 的 `simple-mind-map + RichText + Quill` 基础上补齐行内超链接和完整代码块编辑器，同时扩充真正影响编辑行为的全局设置，并形成后续版本路线。

## 范围

### 行内超链接

- 选中文字后可新增、修改或移除行内链接。
- 支持 `http://`、`https://`、`mailto:`、`tel:`、`siyuan://blocks/`。
- 可选自动为裸域名补充 `https://`。
- 外部链接支持“新窗口”或“当前窗口”打开；思源链接交给思源协议处理。
- 保留原有节点级链接，两者互不替代。

### 完整代码块编辑器

- 富文本工具栏增加行内代码和代码块入口。
- 代码块窗口包含语言、代码内容、换行显示和操作按钮。
- 可把选中文字转换为代码块，也可重新编辑当前代码块。
- 支持移除代码块格式或删除代码块内容。
- 代码块语言写入 Quill 块格式并保存到节点富文本。
- 代码块在节点显示和编辑状态中均有清晰样式。

### 设置

在原有布局、画布和滚轮设置之外增加：

- 自动保存延迟。
- 是否显示富文本选区工具栏。
- 裸链接是否自动补充 HTTPS。
- 外部链接打开方式。
- 默认代码语言、换行、语言标签、Tab 宽度和字号。
- 挖空显示方式及悬停显示答案。
- 是否显示待办和批注状态标记。

设置修改后立即同步到已打开导图；仅影响视图的配置不写入导图正文。

## 架构

- `YeMindRichText` 继承开源 `simple-mind-map` 的 `RichText` 插件，仅扩充 Quill 允许格式，不重写导图内核。
- `inlineLink.ts` 负责链接标准化和协议判断。
- `codeBlock.ts` 负责代码块范围识别、插入、替换和移除，UI 不直接拼装 Quill Delta。
- `richTextDialogs.ts` 负责行内链接和代码块小窗口。
- `SettingsStore` 作为唯一设置状态源；编辑器通过订阅应用 CSS 变量、数据属性和行为配置。
- 节点状态标记使用可配置的模块级显示策略，避免把显示偏好写入每个节点。

## 兼容性

- v0.3.0 数据无需迁移，新增设置缺失时采用默认值。
- 原有节点级链接、公式、挖空、批注、待办、标签、图标、图片、概要和关联线继续可用。
- 新富文本格式由 Quill HTML 保存，导图重新打开后应保持。

## 不在本版范围

- 代码语法高亮依赖的完整语言包。
- 多文件代码编辑器或代码执行。
- 行内链接卡片预览。
- 思源块引用选择器。
