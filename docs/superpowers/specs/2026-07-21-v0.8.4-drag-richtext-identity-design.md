# YeMind v0.8.4 拖动后富文本编辑身份与几何设计

## 问题

节点经过结构拖动（包括最终落回原位置）后，双击编辑会先正确打开，随后一次重定位把编辑框移动到 `-6px/-4px`。诊断证明上游隐藏 SVG 文字节点返回零矩形，旧实现仍把零矩形作为权威坐标。

## 根因

1. 富文本插件以易失的渲染节点实例作为编辑身份；结构重绘后该实例或其 SVG 文字包装器可能失效。
2. `openRealtimeRenderOnNodeTextEdit` 会隐藏原 SVG 文字；隐藏或重绘中的元素可能返回全零 `getBoundingClientRect()`。
3. 上游 `updateTextEditNode()` 不验证矩形有效性，直接写入 `left=0/top=0`，叠加 `-6/-4` 内边距后产生漂移。
4. 完成编辑时仍可能把文字提交给拖动前的旧节点实例。

## 设计

- 编辑事务以节点 UID 为稳定身份。
- 打开、聚焦、重定位和提交前通过 `renderer.findNodeByUid()` 获取当前实例。
- 位置解析按以下优先级：
  1. 当前 SVG 文字元素的有效 `getBoundingClientRect()`；
  2. `getScreenCTM()` 与本地文字尺寸重建的屏幕矩形；
  3. 进入编辑时捕获的最后有效屏幕锚点。
- 宽或高小于等于 0.5px、非有限数值或脱离 DOM 且无可用矩阵的矩形一律无效。
- 无有效新几何时保持当前编辑框位置，绝不回落到页面原点。
- 结束编辑时提交到当前渲染节点。

## 不变契约

- 新节点/默认节点首次双击自动全选。
- 已编辑节点双击光标落在末尾。
- Ctrl/Cmd+A、复制、剪切、粘贴、Backspace、Delete、方向键和中文输入法保持文本语义。
- 分屏和大纲逻辑不受影响。
