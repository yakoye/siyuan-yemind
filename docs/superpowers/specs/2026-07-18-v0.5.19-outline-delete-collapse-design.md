# YeMind v0.5.19 大纲删除、折叠与层级引导设计

## 目标

修复四条真实桌面端回归：空白大纲行无法删除、折叠状态与导图节点展开状态串联、层级不易辨认、画布 Delete/Backspace 进入上游 Root 删除错误对话框。

## 根因证据

1. `resolveOutlineKeyAction()` 对空白行的 Backspace/Delete 固定返回 `none`，因此 Quill 得到空文档删除键后不会产生结构命令。
2. 大纲 `expanded` 直接读取 `node.data.expand`，点击大纲三角又调用 `SET_NODE_EXPAND`；这把“大纲显示折叠”和“导图节点折叠”错误地当成同一状态，Root 折叠后两边同时只剩根节点。
3. 大纲缩进只依赖空白 padding，没有纵向层级引导，深层节点难以追踪父子关系。
4. Delete/Backspace 仍主要由 `simple-mind-map` 的全局 KeyCommand 执行；只要活动选择中包含 Root，上游就可能走到多 Root 保护对话框。安全删除应在 `beforeShortcutRun` 边界统一接管并过滤 Root。

## 方案比较

### 方案 A：继续复用 `node.data.expand`

优点是改动小。缺点是无法消除大纲与画布折叠耦合，正是本次故障根因，不采用。

### 方案 B：大纲维护编辑器局部折叠集合（采用）

每个 `YeMindEditor` 使用 `Set<uid>` 保存仅影响大纲显示的折叠项；渲染、结构签名和 patch 都显式接收该集合。结构改变时清理已不存在的 UID。画布展开状态继续完全归上游维护。

### 方案 C：新增持久化的大纲视图状态字段

可跨重启保存折叠状态，但会改变文档或设置格式，并引入迁移。本轮没有持久化要求，违反 YAGNI，不采用。

## 删除语义

- 大纲编辑器：仅当正文为空、不是 Root、没有组合键且不处于 IME 组合输入时，Backspace/Delete 删除当前节点。正文非空时完全交给 Quill。
- 删除空白行前销毁当前 Quill 会话但不再次写回；删除后聚焦上一条可见行，若没有则聚焦下一条或父节点。
- 画布：上游 Delete/Backspace 一律被安全适配层接管。活动节点中有普通节点时只删除普通节点；只有 Root 时安静阻止，不打开 KMind Root 错误对话框。

## 折叠语义

- 大纲三角只控制大纲可见行，不写 `tree.data.expand`，不调用 `SET_NODE_EXPAND`。
- Root 和普通父节点都可反复折叠/展开；父行始终保留三角。
- 画布上的折叠按钮、子节点数量和导图历史继续由 `simple-mind-map` 管理。

## 层级引导

使用行伪元素绘制类似 VS Code indent-rainbow 的低对比度四级循环竖线。竖线区域宽度由 `--ymz-outline-depth` 控制，不增加额外 DOM，也不拦截鼠标事件。

## 验证范围

- 空白普通节点 Backspace/Delete 删除，Root 不删除，非空正文只删字符。
- 大纲折叠不改变 `data.expand`，Root 可折叠后再次展开。
- 折叠集合在节点删除后自动清理。
- 画布普通节点、Root-only、Root+普通节点三种选择的安全删除决策。
- 层级引导 CSS、完整测试、TypeScript、生产构建、ZIP 和解压后二次验证。
