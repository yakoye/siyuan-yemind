# YeMind Zen v0.7.0 关于与诊断回归设计

## 目标

1. 设置中新增可长期维护的“关于”分区。
2. 把版本信息从分散常量升级为可检测的发布身份。
3. 让用户在 Windows SiYuan 中复现问题后，导出足以定位全局搜索链路的诊断包。
4. 默认保护隐私，不导出完整导图正文。

## 方案比较

### 仅在 README 写版本

无法判断安装包、运行代码和缓存是否一致，也不能服务真实运行诊断，不采用。

### 导出一份巨大 diagnostics.json

实现简单，但人工定位困难、时间线与状态混杂，不采用为唯一格式；保留它仅用于兼容。

### 关于页 + 版本一致性 + 结构化诊断包

采用。关于页用于用户自查，版本门禁用于发布，独立 JSON/JSONL/Markdown 文件用于快速阅读，旧汇总文件用于兼容。

## 数据与隐私

- 日志记录节点 UID、数量、状态、步骤和耗时，不记录查询正文。
- 查询仅记录长度和哈希。
- 默认排除导图标题、节点正文、备注、批注、链接、图片和私人路径。
- 用户明确启用时才增加 `maps-with-content.json`。
- DOM 快照为合成结构状态，不复制真实搜索结果文本。

## 全局搜索事件链

查询变化 → 原生/YeMind 结果数量 → 列表挂载 → 预览挂载/可见 → 结果选中 → Enter 捕获 → 关闭搜索请求/结果 → 打开已有或新建标签 → 编辑器就绪 → 找到节点 → 展开祖先 → 选中 → 居中 → 高亮或失败。

## 兼容性

不修改导图或检查点存储版本；新增诊断 schema v3 和设置 UI，不需要迁移用户数据。
