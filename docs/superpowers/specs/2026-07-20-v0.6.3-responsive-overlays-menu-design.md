# YeMind Zen v0.6.3 响应式浮层、节点内容与菜单设计

日期：2026-07-20

## 目标

本版集中解决小画布和节点内容交互中的一致性问题：

1. 底部“选/拖”和设置页统一改为鼠标箭头/手型图标；
2. 备注、批注悬停预览自动选择不遮挡触发图标的位置；
3. 批注预览显示右对齐的小号日期时间；
4. 顶部和底部工具栏在窄画布中保持单行、可滚动且不挤压换行；
5. 备注与批注对话框统一为紧凑的内容卡片样式；
6. 顶部与节点右键菜单使用同一个“节点样式”入口和图标；
7. 节点右键菜单只显示一个滚动条，并按功能分组整理。

## 方案比较

### 方案 A：只补 CSS

优点：改动小。

缺点：无法可靠避免悬停预览遮住图标，也不能保证设置页与底部模式切换拥有同一交互语义；右键菜单双滚动条根因也不只是颜色和宽度。

结论：不采用。

### 方案 B：替换为完整官方 KMind UI

优点：交互接近官方。

缺点：官方 0.34.0 使用私有注册表、状态树和菜单渲染内核，直接替换会破坏 YeMind 的 simple-mind-map 命令、检查点和设置存储边界。

结论：不采用。

### 方案 C：复用现有命令层，重构展示组件

- 悬停预览增加独立的可测试定位算法；
- 选择/拖动模式共用一套 SVG 图标和展示模型；
- 节点样式只保留一个 NodeStylePanel，顶部和右键菜单均调用它；
- 右键菜单按原有命令分组，不改变命令本身；
- 工具栏只调整响应式容器，不改变已有入口和快捷键。

优点：满足需求，同时保护既有功能、历史和持久化。

结论：采用。

## 详细设计

### 1. 选择/拖动模式

新增统一图标函数：

- `select`：标准鼠标箭头；
- `pan`：张开手掌。

底部按钮只显示图标，通过 title/aria-label 解释完整模式。设置页不再使用纯文字原生 select，改为两个紧凑单选按钮，图标和文案分别为：

- 箭头 + 选择优先；
- 手型 + 拖动优先。

设置值仍使用原有 `canvasMode: 'select' | 'pan'`，不迁移数据。

### 2. 悬停预览定位

新增纯函数 `computeHoverPreviewPlacement`：

- 候选顺序：右下、右上、左下、左上、右、左、下、上；
- 每个候选必须与触发图标保留至少 8px 间距；
- 优先选择完整落在编辑器可视区域内且不与触发图标相交的位置；
- 所有候选都放不下时，选择溢出面积最小的位置并夹紧到画布内；
- 元素写入 `data-placement`，便于诊断和测试。

### 3. 批注预览

每条批注预览包含：

- 正文；
- 右侧小号日期时间；
- 时间使用已有 `formatCommentTimestamp`，避免出现两套格式。

### 4. 小画布工具栏

顶部、底部工具栏：

- 单行 `flex-wrap: nowrap`；
- 子项 `flex: 0 0 auto`，禁止文字拆行；
- 超宽时仅工具栏自身横向滚动；
- 隐藏滚动条但保留滚轮和触控滚动；
- 不改变画布尺寸，也不让工具栏越过当前编辑器边界。

在更窄宽度中隐藏非关键状态文字：保存状态、统计信息；功能按钮仍可滚动访问。

### 5. 备注与批注对话框

统一使用紧凑头部、内容区和底部操作区：

- 关闭按钮固定在内容面板右上角；
- 备注编辑区占据剩余高度；
- 说明文字缩短并弱化；
- 批注卡片正文在上，时间在左下，编辑/删除在右下；
- 新增输入框与“清空全部/添加”底部对齐；
- 对话框宽度对窄屏使用 `min(500px, calc(100vw - 24px))` 的等价运行值。

### 6. 节点样式统一

顶部和右键菜单都调用同一个 `NodeStylePanel.show()`。右键菜单使用与顶部完全相同的 `nodeStyleIcon()`，不再使用宿主 `iconTheme`。

### 7. 节点右键菜单整理

保留高频操作直接显示：

- 编辑节点；
- 添加子/同级/父节点。

其余内容按子菜单分组：

- 剪贴板；
- 节点内容；
- 样式与关系；
- 排列与折叠。

删除操作仍放在底部直接显示，避免误操作被隐藏。菜单外层不滚动，仅 `.b3-menu__items` 负责滚动，从而只出现一列滚动条。

## 兼容性

- 不改变导图数据格式；
- 不改变 canvasMode 设置值；
- 不改变节点内容、样式、概要和关系命令；
- 不改变快捷键；
- 旧设置无需迁移。

## 验收标准

- 底部和设置页模式图标一致；
- 预览不遮挡当前备注/批注图标；
- 批注预览时间可见、右对齐；
- 400px 左右画布宽度下工具栏不换行、不竖排；
- 备注/批注界面紧凑且关闭按钮位于内部；
- 两个节点样式入口打开同一面板；
- 节点菜单只有一个滚动条，分组清晰；
- 全量测试、TypeScript、生产构建与发布 ZIP 验证通过。
