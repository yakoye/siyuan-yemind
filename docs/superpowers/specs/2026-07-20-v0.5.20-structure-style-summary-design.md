# YeMind Zen v0.5.20 结构、样式与概要设计

日期：2026-07-20

## 目标

在不建立第二套导图结构、历史或持久化内核的前提下，完成：

1. 节点尾端快速新增/删除控制；
2. 导图、大纲与分屏折叠状态双向同步；
3. Root 一级分支可折叠；
4. 顶部删除入口移除；
5. 结构、主题与线型工具图标化；
6. 结构、主题与线型仅加入空白画布右键菜单；
7. 单节点样式面板；
8. 原生概要创建、编辑与删除入口。

## 官方源码研究边界

官方 KMind Zen 0.34.0 使用共享编辑器内核提供节点样式、概要、主题和线型。YeMind 继续采用“交互与事务机制适配”方案：

- 最终节点结构继续由 `simple-mind-map` 命令处理；
- 概要继续使用原生 `ADD_GENERALIZATION` 与删除命令；
- 节点样式写入原生节点数据字段并通过原生命令提交；
- 不复制官方私有文档模型、React 状态树或历史内核。

## 方案比较

### 方案 A：直接嵌入官方完整编辑器

优点：外观和行为最接近官方。

缺点：需要引入官方私有数据、历史、渲染和宿主集成，破坏 YeMind 已有存储与检查点边界。

结论：不采用。

### 方案 B：完全自制第二套结构和样式渲染

优点：控制自由。

缺点：与上游节点命令、拖拽、撤销和概要结构产生双轨状态，回归风险最高。

结论：不采用。

### 方案 C：移植官方交互，调用上游命令

优点：满足界面需求，同时保留现有数据、历史、拖拽和保存。

缺点：需要为上游不同版本做适配和充分测试。

结论：采用。

## 设计

### 1. 节点尾端快速控制

使用编辑器局部 SVG 浮层，而不是修改节点正文 HTML：

- 所有可见节点在尾端显示快速控制；
- 叶节点只显示 `+`；
- 非叶节点显示 `−` 与 `+`；
- `+` 调用原生插入子节点；
- `−` 调用安全删除当前节点及子树；
- Root 不显示删除；
- 控件随节点重绘、缩放、移动和布局更新重新定位；
- 控件尺寸小于旧展开按钮，不遮挡文字或内容徽标。

### 2. 折叠状态同步

取消“大纲局部折叠集合”作为权威状态，统一读取节点原生 `expand`：

- 大纲三角调用 `SET_NODE_EXPAND`；
- 导图折叠事件触发大纲重新渲染；
- Root 自身保持可见，Root 的一级分支可以各自折叠；
- 大纲根据节点真实 `expand` 决定子行可见性；
- 历史、保存和重新打开均沿用原生数据。

### 3. 顶部工具栏与空白菜单

顶部：

- 移除“删除”；
- 布局入口显示结构图标和“结构”；
- 主题入口显示主题图标和“主题”；
- 线型入口显示当前线型 SVG，不再显示“弧线”文字。

空白右键菜单：

- 增加结构、主题、线型三个子菜单；
- 节点右键菜单不出现这三项；
- 选择结果与顶部入口保持同步。

### 4. 单节点样式

选中普通节点后打开右侧轻量面板。面板提供：

- 形状：圆角矩形、矩形、菱形、椭圆、胶囊；
- 填充颜色和清除；
- 边框颜色、线型、宽度和清除；
- 节点内容宽度与适应；
- 字体、字号、字重和文字颜色；
- 粗体、斜体、删除线、下划线；
- 左、中、右对齐；
- 重置为主题样式。

规则：

- 多选时批量应用共同操作；
- Root、概要和普通节点均回显真实可用字段；
- 样式操作进入上游历史；
- 不把局部样式写入全局主题。

### 5. 概要

继续使用原生概要：

- 连续同级多选后可添加概要；
- 单节点允许添加覆盖该节点的概要；
- 右键菜单使用专用概要 SVG；
- 概要创建后进入文字编辑；
- 选择概要后可编辑或删除；
- 大纲中概要作为独立结构行显示；
- 节点拖动、删除、折叠后由上游负责范围和布局。

## 兼容性

- 不改变 `MapStorageDocument.version`；
- 旧图没有局部样式时继续继承主题；
- 已有原生概要数据继续兼容；
- v0.5.19 的安全删除、IME、拖拽和检查点契约必须保持。

## 验收标准

- 快速按钮在 Root、普通分支和叶节点上符合显示规则；
- 导图与大纲折叠双向一致，重新打开后保持；
- 顶部无删除入口；
- 空白菜单有结构/主题/线型，节点菜单没有；
- 节点样式可应用、撤销、重做和持久化；
- 概要可添加、编辑、删除并随结构变化稳定；
- 完整测试、TypeScript、生产构建和发布 ZIP 验证通过。
