# YeMind v0.5.16 官方交互对齐说明

日期：2026-07-18

## 对齐范围

本版依据用户提供的 KMind Zen 0.33.0 开源生产包，处理节点表面装饰、颜色面板、删除语义、画布/节点菜单、禅模式退出和节点图片输入。官方私有文档模型不进入 YeMind；结构、历史、布局、图片渲染和持久化继续由 `simple-mind-map` 与现有仓库负责。

## 官方机制与 YeMind 适配

| 官方机制 | v0.5.16 适配 | 保持的边界 |
| --- | --- | --- |
| 画布和节点分别拥有上下文菜单 | 使用两套思源原生 `Menu`，画布菜单只放整图/视图动作 | 不复制官方命令总线 |
| 节点操作集中在长菜单 | 移除三点入口，保留完整节点右键菜单并增加窄滚动 | 不删除既有节点能力 |
| 批注作为稳定节点入口 | 节点表面只显示批注图标 | 数量与正文仍在批注对话框 |
| 颜色通过面板选择 | 52 色、重置默认、系统更多颜色、HEX/RGB 当前值 | 不写入伪默认白色/透明色 |
| 禅模式状态可见且可退出 | 默认收起的右上角胶囊，hover/focus 展开 | 继续调用现有 `toggleZen` |
| 图片支持粘贴和尺寸调整 | 粘贴到当前节点、拖入命中节点，自然尺寸写入 | 不引入官方资产系统 |
| 图片按原始比例调整 | 继续使用上游 `NodeImgAdjust` | 不实现第二套缩放几何 |

## 画布菜单

画布右键菜单提供：定位到中心主题、适配全部节点、重置缩放与位置、整理布局、展开全部节点、折叠全部节点、进入/退出禅模式、进入/退出只读模式。它不包含依赖节点目标的编辑动作，也不创建“第二个根节点”。

## 节点菜单

节点菜单继续提供编辑、添加父/子/同级、复制/剪切/粘贴子树、待办、批注、标签、图标、链接、图片、公式、行内链接、代码块、概要、外框、关联线、移动、展开折叠、整理布局和两种删除语义。只增加展示层 class、最大高度与细滚动条。

## 图片输入与缩放

- 粘贴：当前存在主节点且非只读时，读取剪贴板中的第一张图片。
- 拖入：把客户区坐标转换为导图坐标，命中普通节点或概要节点后写入。
- 初始数据：Data URL、文件名、自然宽度、自然高度、`custom:false`。
- 调整：仍由上游 `NodeImgAdjust` 读取原图比例并成对更新宽高。
- 失败：读取或测量失败显示本地错误消息，不写入节点。

## 数据兼容性

- 插件 ID 不变：`siyuan-yemind-zen`。
- 导图、检查点与设置文件名不变。
- 不增加正式数据 schema 版本。
- 旧设置中的 `showNodeMenuButton` 作为未知字段被忽略。
- 图片继续使用上游节点 `image` / `imageSize` 数据。

## 需要桌面验收

自动化环境不能代替 Windows SiYuan 3.7.2 的真实鼠标、系统剪贴板与原生菜单。安装后重点验证：长节点菜单滚动、空白右键位置、禅胶囊 hover/focus、中文输入法下删除键、本地图片 Ctrl+V、不同缩放/平移状态下图片拖入命中、图片调节点等比例缩放。
