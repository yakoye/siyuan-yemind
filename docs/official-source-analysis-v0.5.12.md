# YeMind v0.5.12：KMind Zen 官方构建源码研究与方案 C 映射

日期：2026-07-17

## 1. 研究对象

本次研究对象是用户提供的 `siyuan-kmind-zen(3).zip`：

- 插件 ID：`siyuan-kmind-zen`
- 显示名称：KMind Zen 思维导图插件
- 版本：`0.33.0`
- 作者：`suka233`

压缩包包含完整可运行的 `index.js` 和 `index.css`，因此它确实是官方实际运行实现，而不只是 README 或截图。需要说明的是，包内 JavaScript 是生产构建后的单体 bundle；经格式化后可以追踪函数和调用链，但不是按原始 TypeScript/React 模块目录公开的工程源码。

## 2. 为什么采用方案 C

### 方案 A：整体嵌入官方运行时

会同时引入 KMind Zen 自己的文档模型、编辑器状态、历史、布局、拖拽和持久化，与 YeMind 当前的 `simple-mind-map`、`maps.json`、检查点和标签恢复形成两套内核，因此否决。

### 方案 B：从 bundle 切出大纲或拖拽模块

这些模块直接依赖官方内部目标类型、编辑事务、场景快照和状态容器。强行切出需要维护脆弱的 bundle 私有符号映射，官方重新构建后模块编号和闭包关系可能变化，因此不作为长期方案。

### 方案 C：移植真实交互机制，继续使用当前内核

采用：

- 研究官方真实状态机、时序和参数；
- 在 YeMind 中重新实现等价的交互控制层；
- 文字、结构、历史、布局和保存仍交给当前 `simple-mind-map`；
- 不导入官方私有数据模型、历史栈、持久化或布局引擎。

## 3. 官方大纲实现的真实机制

官方大纲不是每个节点一个孤立输入框，而是持续存在的 Tiptap/ProseMirror 编辑器。核心按键处理位于格式化 bundle 约第 149003–149175 行。

### 3.1 输入法保护

按键处理第一步检查 `event.isComposing`。中文、日文等输入法仍处于候选组合阶段时，不执行 Enter、Tab、删除或折叠等结构命令。

### 3.2 Shift+Enter

约第 149056–149075 行：

- 普通节点内容执行 hard break；
- 代码块中插入换行；
- 不创建新节点。

### 3.3 Enter

约第 149091–149128 行：

1. 阻止浏览器默认行为；
2. 验证当前编辑状态可提交；
3. 比较当前内容与已提交内容；
4. 内容变化时先提交当前节点；
5. 调用结构创建；
6. 保存待恢复目标；
7. 在编辑器更新后按目标和 `end` 位置恢复光标。

这说明“先提交、后结构、再恢复焦点”是流畅度的关键，而不是单纯给 Enter 绑定一个插入命令。

### 3.4 Tab / Shift+Tab

约第 149130–149139 行：

- Tab：当前节点缩进；
- Shift+Tab：当前节点提升一级；
- 命令完成后重新把焦点放回原节点开头。

### 3.5 删除

约第 149140–149155 行：

- 多选时先提交其他正在编辑的目标，再删除目标集合；
- 单个空节点执行节点删除；
- 结构层负责决定删除后的有效焦点目标。

### 3.6 左右方向键

约第 149157–149174 行：

- 光标位于节点开头且节点已展开时，ArrowLeft 折叠；
- 光标位于节点末尾且节点已折叠时，ArrowRight 展开；
- 操作后分别恢复到开头或末尾。

## 4. 官方拖拽实现的真实机制

### 4.1 每帧合并指针移动

官方节点拖动循环位于约第 138524–138610 行：

- 指针事件只更新 `pendingMove`；
- 同一时刻最多保持一个 `requestAnimationFrame`；
- 每帧读取最新坐标、计算 ghost、候选意图和稳定意图；
- 如果帧执行期间又有新移动，再安排下一帧。

这避免了每个 `pointermove` 都执行完整命中测试和绘制。

### 4.2 60 ms / 3 帧稳定器

约第 136508–136543 行定义：

- 最短稳定时间：60 ms；
- 最少连续帧数：3；
- 候选与当前稳定目标相同，清空 pending；
- 候选变化时重新从第 1 帧计数；
- 时间和帧数同时满足后才切换稳定目标。

这会过滤节点边缘附近一两帧的来回抖动。

### 4.3 父节点贝塞尔引导线

约第 105218–105275 行：

- 成为子节点时，曲线连接目标节点与拖动 ghost；
- 成为同级节点时，曲线连接该同级所属父节点与 ghost；
- 横向布局使用父节点右侧中心到 ghost 左侧中心的三次贝塞尔曲线；
- 纵向布局使用父节点底部中心到 ghost 顶部中心的三次贝塞尔曲线；
- 控制点固定向外偏移 40 px。

### 4.4 原父节点提示

同一区域计算原父节点曲线样式：

- 距离按 140 px 归一化；
- 线宽：`2 - 1.1 × ratio`；
- 透明度：`0.3 + 0.6 × ratio`；
- 近处较粗但较淡，远离后变细但更明显。

## 5. YeMind 的方案 C 映射

### 5.1 大纲事务

`src/editor/YeMindEditor.ts`：

- 文字提交时设置 `outlineTextCommitUid`；
- `data_change` 在文字提交阶段只更新输入框高度，不重建整棵大纲 DOM；
- 结构命令使用 `outlineStructureBusy` 串行化；
- 结构变化前保存 `pendingOutlineFocus`；
- 重建后按 UID 恢复 `start`、`end`、`select-all` 或精确 selection range。

### 5.2 键盘语义

`src/editor/outline.ts` 负责纯动作解析，`YeMindEditor` 负责提交和命令事务：

- IME：`event.isComposing` 与 composition 生命周期双重保护；
- Enter：同级，根节点为子级；
- Shift+Enter：textarea 原生换行；
- Tab：缩进；
- Shift+Tab：提升一级；
- 空 Backspace/Delete：删除并回到上一行，上一行不存在时回到下一行；
- ArrowUp/Down：提交后移动相邻可见行；
- ArrowLeft/Right：光标边界折叠和展开；
- Escape：恢复本次编辑前的文字。

### 5.3 上游命令所有权

`src/core/commands.ts` 继续调用：

```text
SET_NODE_TEXT
INSERT_NODE
INSERT_CHILD_NODE
MOVE_NODE_TO
MOVE_UP_ONE_LEVEL
REMOVE_NODE
SET_NODE_EXPAND
GO_TARGET_NODE
```

没有直接修改 `MindMapTree.children`，也没有第二套撤销历史。

### 5.4 分屏

`src/editor/splitPane.ts` 与 `YeMindEditor`：

- 默认大纲宽度比例 42%；
- 有效范围 25%–70%；
- pointer move 按动画帧合并；
- 方向键每次调整 2%；
- 双击或 Home 恢复默认值；
- 写入 `SettingsStore.splitOutlineRatio`；
- 只在分屏可见且画布有效时安全 resize。

### 5.5 拖拽

`src/core/YeMindDrag.ts`：

- 继承安装版本 `simple-mind-map` 的 `Drag`；
- 保存上游原始 `checkOverlapNode`；
- 把上游固定 300 ms 的调用节流替换为一帧一次的调度；
- 候选目标仍由上游原始命中算法产生；
- 使用 60 ms / 3 帧状态机决定稳定候选；
- 最终 mouseup 前把稳定候选写回上游字段；
- 最终移动仍由上游 `MOVE_NODE_TO`、`INSERT_BEFORE`、`INSERT_AFTER` 完成；
- 目标父节点绘制绿色 `6 6` 贝塞尔虚线；
- 尚无稳定新目标时绘制红色 `3 6` 原父节点曲线。

## 6. 明确没有复制的内容

v0.5.12 没有导入或打包 KMind Zen 的：

- 单体 `index.js`；
- React/Tiptap 编辑器运行时；
- 文档目标类型和 scene snapshot；
- 导图数据模型；
- 存储文件；
- 历史栈；
- 布局引擎；
- 拖拽最终结构算法。

本版只把经过源码确认的交互契约、时序、稳定参数和几何规则重新实现到 YeMind 当前架构中。

## 7. 仍需真实桌面端确认

自动化能够验证动作解析、命令桥、稳定器和几何公式，但不能替代 Windows SiYuan 3.7.2 中的真实感受。仍需手工确认：

- 中文输入法候选状态下 Enter 不误建节点；
- 连续 Enter、Tab、Shift+Tab、删除时焦点不跳；
- 长文本和节点内换行的光标恢复；
- 分屏拖动时画布与大纲连续响应；
- 快速跨越节点边缘时父节点曲线不抖动；
- 很快拖动并立即释放时最终落点符合预期；
- 横向、组织结构和目录组织布局的曲线方向正确。
