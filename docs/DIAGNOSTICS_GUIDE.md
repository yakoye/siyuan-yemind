# YeMind 诊断与回归使用说明

## 推荐流程

1. 打开 `设置 → 关于 → 诊断与回归`。
2. 点击“开始记录”。
3. 立即复现问题，避免夹杂无关操作。
4. 问题出现后马上点击“标记问题发生”。
5. 可点击“运行回归检查”查看当前状态，但不要因此清空日志。
6. 点击“导出诊断包”，把生成的 ZIP 提供给开发者。

## 全局搜索问题

复现时建议完整执行一次：

```text
打开全局搜索
→ 输入完整叶子节点内容
→ 观察结果和大纲预览
→ 选中 YeMind 结果
→ 按 Enter
→ 观察搜索窗口是否关闭
→ 观察导图是否打开
→ 观察目标节点是否展开、居中和高亮
→ 标记问题发生
→ 导出诊断包
```

诊断会记录查询长度与哈希、思源/YeMind 结果数量、结果列表和预览挂载状态、选中类型、Enter 捕获、关闭搜索、打开标签、编辑器就绪、展开祖先、居中和高亮步骤。默认不记录查询正文。

## 回归检查的含义

- 存储、导图生命周期、树结构、检查点、设置和编辑器状态会主动检查。
- 全局搜索依赖真实思源窗口和用户操作，因此未复现过时会显示“尚未记录”的警告，而不会给出虚假的通过。
- 复现后会根据实际事件判断结果/预览是否存在，以及最近一次导航成功或失败在什么步骤。

## 隐私

默认诊断包不包含导图标题、节点正文、备注、批注、链接、图片或搜索输入内容。DOM 快照只保留结构、类名、ID、挂载和隐藏状态，导图/节点标识会被替换。

只有排查内容解析问题时才勾选“包含导图标题与节点文字”。勾选后请先确认诊断包可以安全分享。
