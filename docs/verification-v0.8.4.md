# YeMind v0.8.4 验证报告

日期：2026-07-21

## 发布目标

修复节点发生结构拖动或同位拖动后，双击编辑出现空白框、文字漂到页面左上角的问题，同时保证既有编辑、拖拽、折叠、搜索、保存和诊断功能不回归。

## 根因验证

用户诊断包明确显示，编辑框先在正确位置打开，随后被一次零矩形更新移动到 `-6/-4`。源码分析确认结构拖动会更换渲染节点对象，而隐藏 SVG 文字可能返回零尺寸矩形。

本版以稳定 UID、当前渲染实例和有效几何回退链重构编辑定位，没有针对具体错误坐标写补丁。

## 专项集成测试

测试使用真实 `simple-mind-map + SVG + Quill`：

1. 在带偏移的思源式编辑容器中创建真实导图；
2. 双击真实 SVG 节点打开 `YeMindRichText`；
3. 确认编辑框初始位置与节点相符；
4. 模拟同位拖动后留下的失效节点对象；
5. 让隐藏 SVG 返回 `0 × 0` 且无屏幕矩阵；
6. 调用真实重新定位流程；
7. 确认编辑框保留最后有效锚点，绝不移动到 `-6/-4`；
8. 确认编辑事务重新绑定当前 renderer 节点；
9. 模拟隐藏文字仍有有效 `getScreenCTM()`，确认可重建拖动后的新位置；
10. 结束编辑时确认 `SET_NODE_TEXT` 接收当前节点，而非拖动前实例。

## 既有功能回归

完整回归继续覆盖：

- 新节点首次全选、旧节点末尾光标；
- Ctrl/Cmd+A、复制、剪切、粘贴和删除键；
- 分屏与大纲稳定编辑、中文输入法和结构拖动；
- Root 与普通分支折叠展开；
- 右键拖动画布；
- 富文本工具栏、颜色、备注、批注、图片、概要和节点样式；
- 全局搜索预览、Enter/双击打开和节点定位；
- 保存、检查点、插件身份和旧数据迁移。

## 桌面端验收重点

当前构建环境无法直接启动 Windows SiYuan。真实桌面端安装后重点验证：

1. 任意节点轻微拖动后回到原处，再双击编辑；
2. 节点跨层级或同级换位后，再双击编辑；
3. 连续拖动多个节点后，分别进入编辑；
4. 新节点仍自动全选，旧节点仍落在末尾；
5. 输入、Ctrl+A、复制、剪切、粘贴和中文输入法正常；
6. 编辑后保存、关闭、重新打开，文字与结构正确；
7. 诊断日志不再出现有效 `opened` 后紧跟 `hostLeft=-6 / hostTop=-4`。
