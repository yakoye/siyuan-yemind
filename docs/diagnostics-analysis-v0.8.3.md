# YeMind v0.8.3 用户诊断包分析

诊断包：`yemind-diagnostics-20260721-192819.zip`

## 观察结果

- 诊断生成版本：0.8.1；
- 版本一致性：通过；
- 共记录 189 个事件；
- 全局搜索并非本次复现目标；
- 大纲多次记录 `edit-start` 后由 `structure-change` 销毁编辑器；
- 旧版没有画布 RichText 的宿主位置、节点位置和初始选区事件。

## 根因定位

源码检查发现，上游 RichText 编辑宿主采用固定定位和视口坐标，但 YeMind 通过 `customInnerElsAppendTo` 将它挂到思源标签内的编辑器容器。思源标签布局具有独立的定位上下文，因此视口坐标被错误解释为容器坐标，造成：

- 原节点位置只留下空白编辑框；
- 实际文字编辑层漂到页面左上方或其他位置；
- 用户看到的节点与真实可编辑文字不在同一处。

此外，上游 UMD 已预注册默认 RichText，旧注册逻辑可能跳过 YeMind 同名覆盖，使测试和生产行为不一致。

## v0.8.3 修复

- 编辑框改用编辑器局部绝对坐标；
- 实时渲染更新后重新定位；
- 首次节点全选、已有节点末尾光标；
- 画布和大纲本地处理 Ctrl/Cmd+A；
- 注册器强制替换同名 RichText；
- 诊断增加编辑框位置、节点位置和选区策略事件。
