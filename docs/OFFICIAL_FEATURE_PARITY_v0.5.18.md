# YeMind Zen v0.5.18 官方功能对齐边界

| 主题 | KMind Zen 0.34.0 | YeMind Zen v0.5.18 | 保留边界 |
|---|---|---|---|
| 大纲编辑器 | 连续 ProseMirror/Tiptap 文档 | 单活动 Quill + 不移动活动行的 UID keyed patch | 不引入官方私有 schema/文档树 |
| 普通文字同步 | 编辑事务内更新稳定块 | 文字提交后模型回流不重插活动行 | 节点正文仍用上游 `SET_NODE_TEXT` |
| IME/selection | 连续编辑器事务保持 | 活动 Quill 会话保持；组合输入期间不执行结构命令 | 不增加第二个 selection 模型 |
| 提交生命周期 | 命令总线串行事务 | committing guard + `commitAndDetach(reason)` | 自动保存仍使用现有仓库 |
| 大纲拖拽 | 指针几何、层级和 drop slot | 整行 Pointer Events、长按正文保护、X/Y 层级解析 | 最终移动仍由上游 MOVE/INSERT |
| 拖拽历史 | 官方文档历史 | `simple-mind-map` 原生历史 | 不直接写树 |
| 折叠/展开 | 独立实体属性 | 最新状态 resolver、不可见焦点清理、叶节点占位 | 展开数据继续是现有 `expand` |
| 公式入口 | 数学公式工具 | 数学字体粗体 `π` 和无障碍标签 | 公式数据格式不变 |
| 模糊/Cloze | 编辑器装饰 | 兼容旧格式的高斯/毛玻璃显示层 | 旧节点数据无需迁移 |
| 思源原生大纲 | 基于块模型和 Protyle | 仅采用稳定事务与拖拽交互原则 | 不调用不稳定的思源内部块编辑 API |

## 本版结论

v0.5.18 移植的是官方稳定大纲的事务原则和指针落点语义，而不是复制官方私有编辑内核。这样既解决焦点丢失、拖动误清空和折叠状态竞争，又继续让 `simple-mind-map` 负责节点结构、布局、历史与最终持久化。
