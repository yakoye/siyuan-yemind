# YeMind v0.5.1 验证记录

验证日期：2026-07-16

## 自动化验证

- `npm test`：28 个测试文件、68 项测试全部通过。
- `npm run check`：TypeScript `tsc --noEmit` 通过。
- `npm run build`：Vite 生产构建通过，转换 839 个模块。
- `node --check index.js`：生成的插件入口语法检查通过。
- `plugin.json`：插件 ID 为 `siyuan-yemind-zen`，显示名为 `YeMind`，版本为 `0.5.1`。

## 已覆盖回归场景

- 结构化模式默认关闭单节点自由坐标。
- 结构化模式默认关闭拖拽边缘自动平移。
- 节点树中的历史 `customLeft/customTop` 会递归清除，节点父子结构保持不变。
- 二级和下级节点间距正确映射到导图主题布局参数。
- 拖拽过程中不持久化临时画布位置；结束后可恢复拖拽前视图。
- 非法、非有限或极端历史画布坐标不会恢复。
- 空标题导图加载为“未命名导图”，不会被过滤。
- 数据尚在加载时执行新建，会等待加载结束并同时保留旧导图和新导图。
- “整理布局”调用 `simple-mind-map` 原生 `RESET_LAYOUT` 命令。
- 设置窗口包含常规、拖拽与布局、节点与内容、快捷键四个页面。

## 仍需在思源实际界面验证

自动化测试无法替代思源桌面端真实鼠标交互，安装后应重点验证：

1. 拖动包含两级以上子节点的父节点，确认整棵子树跟随。
2. 把节点放到同级节点上方、下方和其他父节点下，确认间距自动重新计算。
3. 在画布四边拖动节点，确认默认设置下画布不会突然飞走。
4. 重启思源，确认“未命名导图”仍显示在 Dock 中。
5. 调整二级、下级节点间距设置，确认已打开导图实时重新排版。
6. 在自由拖拽模式与结构化拖拽模式之间切换，确认回到结构化模式时自由坐标被清理。
