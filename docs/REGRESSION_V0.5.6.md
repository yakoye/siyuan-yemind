# YeMind Zen v0.5.6 回归检查与稳定性修复报告

本版不增加新功能，只检查和修复 v0.5.5 及以前已经存在的功能。检查方式包括源码路径复核、上游命令边界检查、失败场景测试、TypeScript 检查、生产构建和安装包解压复检。

> 当前环境无法启动用户本机的思源 v3.7.2 并执行真实鼠标操作，因此“通过”表示代码级、自动化和构建级回归通过；真实界面操作仍需要安装后做最后一轮人工确认。

## 1. 右键菜单

检查范围：

- 编辑节点
- 添加子节点、同级节点、父节点
- 复制、剪切、粘贴节点子树
- 待办、批注、标签、图标、链接、图片、公式
- 行内链接、代码块
- 概要、关联线
- 上移、下移、展开/折叠、整理布局
- 仅删除当前节点、删除节点和子树

修复：

- 只读模式下所有修改类命令由菜单状态和命令适配器双重阻止。
- 根节点不能被剪切、删除、添加同级/父节点或上下移动。
- 概要节点不再误用普通节点命令。
- 多选后在某个已选节点上右键，会将该节点提升为当前主节点，避免批注、待办、链接等操作落到其他已选节点。
- 删除概要时只删除当前概要，或按菜单状态删除所属节点的全部概要。
- 粘贴失败会显示错误，不再产生未处理的 Promise rejection。

## 2. 文字选区与富文本工具栏

检查范围：

- 加粗、斜体、下划线、删除线
- 文字颜色、背景色
- 字号、字体
- 清除格式、挖空
- 公式、行内链接、行内代码和代码块

修复：

- 补全 Quill 字体和字号 Attributor 注册，避免按钮存在但格式无法持久化。
- 点击颜色、字号、字体等工具时保存并恢复当前选区，避免控件抢焦点后格式作用于错误位置。
- 行内链接继续执行协议校验，拒绝 `javascript:` 等不安全地址。

## 3. 节点内容

检查范围：待办、批注、标签、图标、节点链接、图片、公式、概要和关联线。

修复：

- 只读模式下节点待办标记不能被点击修改。
- 批注标记在只读模式下仍可查看，但隐藏新增、编辑、删除和清空操作。
- 节点链接和行内链接统一经过同一套地址规范化与打开方式设置。
- 通过 `simple-mind-map` 的 `customHyperlinkJump` 接管节点链接，避免节点链接始终强制新窗口。
- 本地图片读取失败时清理临时状态并显示错误，不再留下未处理异常。
- 删除节点后继续清理失效关联线，同时保持关联线并行数据索引一致。

## 4. 保存、新建、重命名、删除和关闭

修复：

- 新建导图时标题和默认布局在同一个持久化事务中保存。
- 新建、重命名、修改和删除都先保存草稿；只有写盘成功后才替换已提交的内存状态。
- 保存失败不会让 Dock 和标签页显示一个实际上没有写入磁盘的状态。
- 仓库写入严格串行，较旧的异步保存不能覆盖较新的导图数据。
- 自动保存增加修订号，旧保存完成不会把较新的修改显示为“已保存”。
- 关闭标签页时不仅刷新等待中的计时器；即使上一次保存失败且计时器已结束，只要仍是脏数据也会再次提交。
- 删除导图引发的多次关闭请求变为幂等，同一标签不会重复执行关闭。

## 5. 恢复标签页和退出生命周期

修复：

- 思源恢复自定义标签页时，等待导图仓库和设置加载完成后再创建编辑器。
- 加载期间标签被关闭，不会在异步完成后重新挂载。
- 编辑器创建失败时清理已经登记的标签句柄，并显示加载失败提示。
- 编辑器销毁前先提交当前脏数据快照，再解绑订阅、工具栏和导图实例。

## 6. 设置

检查范围：

- 常规、拖拽与布局、节点与内容、快捷键四页
- 所有 `DEFAULT_SETTINGS` 字段是否都有对应控件
- 快捷键录制、禁用、恢复默认和冲突检查
- 保存、取消、恢复默认、右上角关闭

修复：

- 设置保存改为串行事务；保存失败不会提前修改当前设置。
- 设置窗口只在写盘成功后关闭；失败时保留窗口和用户输入并恢复保存按钮。
- 快捷键录制监听器在取消、保存和右上角关闭时全部清理。
- “重置缩放”改名为“重置视图（100%）”，准确对应上游 `view.reset()` 的缩放和平移重置行为。

## 7. 节点拖动、画布拖动和布局

确认继续使用 `simple-mind-map` 原生路径：

- 节点结构移动：`Drag` 插件
- 父子变更：`MOVE_NODE_TO`
- 同级排序：`INSERT_BEFORE` / `INSERT_AFTER`
- 框选：`Select` 插件
- 多选：原生 active node list
- 布局重算：原生 renderer/layout

YeMind Zen 不再维护第二套拖拽坐标、框选矩形、激活节点列表或拖拽后视图恢复逻辑。

检查项：

- 带子节点的父节点作为完整子树移动
- 多选时仅移动最高层所选子树
- 同时选择父节点和后代不会重复移动
- 平移优先和选择优先两种画布模式
- 拖拽边缘自动平移设置
- 节点间距、画布限制和整理布局

## 8. 放大、缩小、适配和视图重置

确认工具栏入口和命令映射：

- 放大：`view.enlarge()`
- 缩小：`view.narrow()`
- 适配并居中：`view.fit()`
- 重置视图（100%）：`view.reset()`
- 缩放范围：`minZoomRatio` / `maxZoomRatio`
- 适配留白：`fitPadding`

修复：

- 初次打开只由上游 `fit` 配置执行一次适配，删除重复的延时适配。
- 保存的视图数据先校验有限值、缩放范围和合理平移范围，再传给上游恢复。
- 工具栏回归检查覆盖适配、重置、放大、缩小、布局、只读、禅模式和全屏入口。

## 9. 自动化覆盖

本版新增或扩展的专项覆盖包括：

- 完整右键菜单入口和状态
- 右键多选主节点
- 根节点和只读保护
- 富文本全部格式和选区保留
- 节点/行内链接统一跳转
- 本地图片加载失败
- 事务式导图仓库
- 自动保存修订和关闭重试
- 设置字段完整性和写盘生命周期
- 恢复标签页失败清理与幂等关闭
- 节点/画布拖拽上游生命周期
- 缩放、适配、重置和布局工具栏入口

```bash
npm run build
node --check index.js
```
