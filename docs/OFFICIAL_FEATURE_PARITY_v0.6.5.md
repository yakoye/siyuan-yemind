# YeMind Zen v0.6.5 官方交互适配边界

日期：2026-07-20

## 本版范围

v0.6.5 继续采用“移植交互与事务机制、保留 YeMind 数据和历史边界”的方案，完成整图密度重排和自定义间距、样式面板外部关闭、思源搜索延迟定位、节点颜色面板统一，以及画布富文本可见性和局部选区回归强化。

## 已适配

- 整图密度现在分为更紧凑的“紧凑”、沿用旧紧凑参数的“默认”、继承主题原始间距的“舒展”，并提供左右/上下自定义间距。
- 自定义间距通过现有 `projectStyle` 写入导图仓库和检查点，不建立第二套布局配置。
- 整图样式面板监听编辑器局部之外的点击并关闭；面板内部颜色、数字和开关操作不会误关闭。
- 思源搜索结果点击后，若对应导图标签页仍在挂载，目标节点 UID 会进入标签页局部待处理状态；编辑器就绪后再调用现有 `focusNode`。
- 节点填充、边框和文字颜色复用富文本的 52 色、重置默认、原生更多颜色及可编辑 HEX/RGB 交互。
- 画布节点进入上游 Quill 编辑器后，YeMind 从当前节点读取真实文字色和安全编辑背景，并同步到编辑包装器、Quill 容器与正文。
- 真实 SVG 双击测试覆盖进入编辑、文字可见、局部选择、工具栏显示和只对选区加粗。

## 保持不变

- 不引入 KMind 私有文档模型、React 状态树、第二套撤销历史或第二套节点样式数据。
- 不替换 `simple-mind-map` 的节点结构、布局、样式命令和富文本提交路径。
- 节点颜色仍写入上游原生样式字段；共享色板只是统一输入界面。
- 搜索延迟定位只是标签页挂载期间的临时 UID，不写入导图，也不建立隐藏文档。
- “舒展”继续继承所选主题自己的间距，不强行覆盖为固定数值。

## 自动验证边界

自动测试覆盖密度参数、数值归一化、自定义间距保存和检查点、外部点击关闭、搜索目标排队与消费、共享色板、颜色输入隔离，以及真实 SVG/Quill 的双击和局部格式化。真实 Windows SiYuan 3.7.2 中仍需验收不同主题下的实际密度观感、思源搜索 DOM 的点击跳转、原生颜色选择器，以及浅色/深色节点进入编辑后的视觉效果。
