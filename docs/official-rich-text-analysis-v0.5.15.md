# YeMind Zen v0.5.15 官方富文本与界面隔离研究

日期：2026-07-18

## 研究范围

本版针对真实 SiYuan 3.7.2 截图中的七项问题，重新对照用户提供的 KMind Zen 0.33.0 官方运行源码和当前 `simple-mind-map` 0.14.0-fix.3：

1. YeMind 浮层越过思源设置窗口；
2. 双击节点后文字不可见；
3. 文字颜色需要直观色板、更多颜色和吸管；
4. 背景颜色需要同样交互；
5. 删除两个独立的颜色清除“X”；
6. 分屏与纯大纲需要完整富文本选区工具栏；
7. 节点中只选择部分文字后进行格式化。

## 官方实现结论

KMind Zen 的顺滑体验来自三个机制，而不是单个 CSS 补丁：

- 富文本操作始终围绕当前 selection，工具栏只是编辑器事务的控制面；
- 大纲使用持续编辑器和 selection 恢复，不把每次富文本操作降级为纯文本；
- 编辑器弹层属于编辑器局部 surface，不应作为高层全局浮层长期挂在宿主页面最外层。

KMind 的完整编辑器依赖私有文档模型、历史和渲染内核，因此仍采用方案 C：移植交互和事务机制，保留 YeMind 的 `simple-mind-map` 结构、历史、检查点与持久化。

## 问题 1：思源设置被 YeMind 工具覆盖

### 根因

顶部工具栏、左侧工具栏、底部状态栏、搜索层和富文本编辑层处于较高 z-index；部分上游内部元素默认按全局页面定位。没有编辑器局部堆叠上下文时，它们可能参与 SiYuan 整个页面的 z-index 竞争，从而出现在设置窗口上方。

### 修复

- `.ymz-editor` 建立独立 stacking context；
- 所有 YeMind 富文本工具栏和颜色面板挂载到当前 `.ymz-editor`；
- `simple-mind-map` 的 `customInnerElsAppendTo` 指向当前编辑器根节点；
- 不使用会改变 fixed containing block 的 `contain: paint`，避免上游节点编辑框错位；
- 诊断导出产生的临时下载节点不属于持续 UI 浮层，保留短生命周期的 `document.body` 下载行为。

这样 YeMind 内部仍可按自己的层级显示，但整个标签页只作为一个局部层参与 SiYuan 宿主排序。

## 问题 2：双击节点后编辑文字不可见

### 根因

上游 RichText 编辑器创建独立 Quill 编辑层。编辑层在思源全局样式、主题文字色和节点自身颜色之间可能继承到错误的 `color` 或 `-webkit-text-fill-color`，导致浅色节点上出现白字或透明字。

### 修复

对当前编辑器内的上游富文本编辑包装器、Quill 容器、编辑区及内部元素明确设置：

- `color: inherit`；
- `caret-color: currentColor`；
- `-webkit-text-fill-color: currentColor`；
- 选区使用思源主题主色的半透明背景，保持文字可见。

## 问题 3～5：文字颜色与背景颜色

### 用户要求优先于机械复制

官方具有颜色控制和清除入口，但用户提供的色板方案更直观。因此本版保留官方 selection 驱动机制，采用用户指定的交互：

- 52 个常用色块；
- 一个“重置默认”；
- “更多颜色”调用系统原生 `<input type=color>`；
- 运行环境支持 `EyeDropper` 时显示真实屏幕吸管；
- 不支持或吸管取消时安全回退/结束，不产生运行错误；
- 删除文字颜色和背景颜色旁边的两个独立“X”。

重置不是写入白色或透明色，而是向富文本目标发送 `false`，删除当前 selection 的对应格式并恢复继承值。

## 问题 6：分屏和大纲完整富文本

### 旧实现问题

旧大纲以 textarea/纯文本为核心。即使节点原本包含富 HTML，进入大纲编辑也会丢失局部 span 格式，并且没有稳定的富文本 selection 可供工具栏操作。

### 新实现

- 大纲行保留 `text`、`html` 和 `richText` 状态；
- 非活动行渲染经过清理的轻量 HTML；
- 当前编辑行只创建一个活动 Quill 实例；
- 导图、分屏和纯大纲通过统一的 `RichTextFormattingTarget` 使用同一个工具栏；
- 当前大纲 selection 可以设置加粗、斜体、下划线、删除线、行内代码、代码块、文字颜色、背景颜色、字体、字号、链接、挖空、公式和清除格式；
- 文字改变后节流写回，结构操作前立即 flush；
- 失焦、切换节点、切换视图和销毁时提交或按明确语义取消；
- HTML 最终仍通过上游 `SET_NODE_TEXT` 富文本模式写回。

只维持一个活动 Quill，避免大型导图为每个大纲行长期创建编辑器。

## 问题 7：只格式化节点中的几个字

上游配置原来在进入编辑时自动选中整段节点文字。现在设置：

```text
selectTextOnEnterEditText: false
```

用户双击进入编辑后可以自然拖选其中几个字，工具栏读取实际 selection range，只修改该范围。整个节点样式与局部文字样式继续区分。

## 保持不变的内核边界

本版没有引入：

- 第二套导图结构；
- 第二套撤销历史；
- KMind 私有文档模型；
- 新的正式导图仓库；
- 新的检查点格式；
- 每节点一个永久 Quill 实例。

结构命令、导图布局、拖拽落点、历史和持久化继续由现有上游与 YeMind 仓库负责。

## 需要真实桌面端验收的内容

自动化环境不能替代 Windows SiYuan 3.7.2 的真实窗口层级、原生颜色选择器和屏幕吸管。安装后应重点验证：

1. 打开思源设置，所有 YeMind 工具都在设置窗口下方；
2. 浅色、深色和自定义节点上双击编辑，文字和光标均可见；
3. 只选择节点中的几个字，分别设置各种格式；
4. 色板、重置默认、更多颜色和吸管；
5. 分屏/纯大纲的局部选区格式和重新打开后的持久化；
6. 中文输入法、Enter/Tab 等结构操作没有回退。
