# YeMind Zen

YeMind Zen 是一个基于 `simple-mind-map` 构建的思源笔记思维导图插件。

- 插件目录和插件 ID：`siyuan-yemind-zen`
- 思源显示名：`YeMind Zen`
- 当前版本：`0.5.17`
- 本地保存，不包含付费、试用、激活或会员逻辑

## v0.5.17：大纲稳定编辑、备注悬停与可编辑颜色

本版以用户提供的 KMind Zen 0.34.0 官方包为最新交互参考，重点重做分屏/大纲编辑事务，并补齐备注、颜色和诊断反馈链路。

- 大纲按稳定节点 UID 增量更新，当前活动 Quill 行不会因自动保存或导图 `data_change` 被整体替换。
- 中文输入法组合期间暂停结构快捷键和自动提交；清空文字只提交空正文，不再隐式删除节点。
- Enter 在根节点创建子节点、普通节点创建同级节点；Shift+Enter 换行；Tab/Shift+Tab 调整层级；上下键仅在首尾边界跨行。
- 每条大纲行提供独立拖拽把手，可调整同级顺序、拖入成为子节点或拖出层级；最终结构仍由上游 `MOVE_NODE_TO/INSERT_BEFORE/INSERT_AFTER` 和历史栈负责。
- 颜色面板按“色板 → 重置默认/更多颜色 → HEX/RGB”组织；HEX 与 RGB 可编辑、实时双向转换，输入事件不会落到节点正文或结构快捷键。
- 新增长文备注、备注图片粘贴、可拖动调整备注窗口大小，以及备注/批注图标悬停预览；批注右键菜单使用插件自带 SVG 图标。
- 禅模式状态胶囊默认显示“● 禅”，悬停或键盘聚焦显示“● 退出禅模式”。
- 待办右键菜单统一为“添加待办/删除待办”，完成状态只由节点复选框切换。
- 用户界面将“挖空/取消挖空”改为“模糊/取消模糊”，公式按钮从 `Fx` 改为 `π`，底层旧数据保持兼容。
- 诊断日志增加问题时间标记、前后事件窗口、大纲编辑/IME/拖拽事务和版本一致性检查，并聚合高频视图噪声。
- 修复运行时版本常量长期停留在 `0.5.13` 的问题；清单、构建和诊断运行时现统一为 `0.5.17`。

详细研究见 `docs/official-source-outline-notes-analysis-v0.5.17.md`，对齐边界见 `docs/OFFICIAL_FEATURE_PARITY_v0.5.17.md`。

## v0.5.16：节点菜单、图片与禅模式交互

- 移除节点三点菜单和旧设置契约；批注图标不再显示数量。
- 空白画布与节点使用独立右键菜单，节点长菜单支持细窄滚动。
- 支持向当前节点粘贴图片、把本地图片拖到目标节点，并沿用上游等比例缩放。
- 禅模式退出入口改为右上角状态胶囊。

## v0.5.15：富文本选区、颜色面板与宿主界面隔离

本版继续研究 KMind Zen 0.33.0 官方运行源码中的选区工具栏、持续编辑器和弹层挂载方式，同时按用户要求改进颜色交互。

- YeMind 顶部、左侧、底部状态栏、搜索和富文本浮层全部限制在当前导图标签内部，不再越过思源设置等宿主弹窗。
- 修复双击节点时编辑文字继承为白色或透明的问题，编辑态文字、光标和选区明确继承节点当前样式。
- 双击节点不再自动全选，可用鼠标只选择其中几个字，再单独设置加粗、斜体、下划线、删除线、行内代码、颜色、背景、链接、挖空、公式等格式。
- 文字颜色和背景颜色改为直观色板；色板内提供“重置默认”“更多颜色”和“吸管”，移除工具栏中两个独立的“X”。
- “更多颜色”使用系统原生颜色选择器；支持 `EyeDropper` 的运行环境直接提供吸管，不支持时安全回退到原生颜色选择器。
- 分屏与纯大纲使用同一套选区富文本工具栏。大纲仅为当前编辑行创建一个活动 Quill 实例，其余行保持轻量 HTML，兼顾富文本能力和大导图性能。
- 大纲富文本写回继续使用上游 `SET_NODE_TEXT` 富文本模式，结构命令、撤销历史、导图保存和检查点机制不变。

详细研究与迁移说明见 `docs/OFFICIAL_FEATURE_PARITY_v0.5.15.md`。

## v0.5.14：官方主题、弧线与完整布局基础

本版继续基于用户提供的 KMind Zen 0.33.0 官方运行源码进行迁移，不根据截图猜测外观。

- 新建和旧版迁移导图默认使用 KMind Default 与弧线父子连线。
- 顶部可按导图选择 13 个官方同名主题，以及弧线、圆角折线和直线。
- 主题提供明暗变体，并跟随思源当前外观实时切换。
- Rainbow Breeze、Midnight Neon 等主题通过上游 `RainbowLines` 渲染主题彩虹分支。
- 主题和线型进入正式导图存储与检查点，关闭和恢复后保持。
- 布局选择开放当前 `simple-mind-map` 支持的全部 14 种结构。
- 没有导入 KMind 私有文档内核、历史栈或持久化实现。

详细对照见 `docs/OFFICIAL_FEATURE_PARITY_v0.5.14.md`。

## v0.5.13：节点拖动跟手性与旧连线修复

本版继续研究用户提供的 KMind Zen 0.33.0 官方运行源码，修复两项真实鼠标回归：

- 拖动开始后，隐藏父节点持有的旧“父 → 被拖节点”入线；多选拖动时分别处理每个顶层节点。
- 记录旧连线原本是否可见，拖动结束、取消或插件销毁时只恢复原来可见的线。
- 拖动目标不再依赖上游鼠标点和四分之一节点命中区，而是使用完整拖动节点矩形。
- 对齐官方 80 px 子节点尾区、8/22 px 子目标进入/离开留白、44/72 px 同级通道和端部留白。
- 对齐“子节点尾区优先 → 同级通道 → 子节点主体 → 同级兜底”的候选顺序。
- 候选处于等待状态时继续逐帧计算；鼠标停住后也能达到 60 ms/连续 3 帧条件，不必再移动一次才切换。
- 目录组织等混合布局按实际目标父节点选择横向或纵向贝塞尔曲线。
- 最终落点继续由上游 `MOVE_NODE_TO`、`INSERT_BEFORE`、`INSERT_AFTER` 执行，没有第二套结构、历史或保存实现。

## v0.5.12：官方交互机制对齐

- 大纲改为“先提交文字 → 串行结构变更 → 按节点 UID 恢复光标”的编辑事务。
- 中文输入法组合期间不会触发结构快捷键。
- 支持 Enter 同级、根节点 Enter 子级、Shift+Enter 节点内换行、Tab 缩进、Shift+Tab 提升一级。
- 支持空节点删除、上下导航、光标边界折叠/展开和 Escape 取消修改。
- 分屏线可拖动、键盘调整、双击/Home 复位，并保存大纲宽度比例。
- 拖动候选按动画帧计算，使用 60 ms/连续 3 帧稳定器和贝塞尔父节点提示。

## v0.5.11：启动恢复与可编辑大纲

- 插件先启动正式导图、设置和检查点仓库加载，再逐项注册 Tab、Dock、设置、命令和协议事件。
- 单个思源界面注册异常会被隔离并写入诊断，不再让 `ready` 假装已经完成、正式导图库却保持未加载。
- 启动后无需新建导图即可显示原有导图，恢复标签页等待真实仓库就绪后再判断导图是否存在。
- 分屏的大纲位于右侧，纯大纲占满工作区；两者共用同一套编辑逻辑。
- 大纲文字和结构修改继续调用上游命令，不直接改写树数据。

## v0.5.10：真实运行诊断修复

本版基于用户从思源 v3.7.2 导出的两份诊断包修复真实运行问题，不增加新的导图业务功能：

- 诊断生命周期自检改用独立临时仓库，不再把临时导图插入正式 Dock 后再删除。
- Dock 等待导图仓库加载完成后再显示最终列表。
- 恢复标签页等待容器真正可见且宽高有效后再创建导图编辑器。
- 纯大纲模式不再调用零尺寸导图画布的 `resize()`。
- 陈旧标签句柄会被清理，不能阻止同一导图重新打开。
- 待办复选框只在未完成和已完成之间切换；删除待办只保留在右键菜单。
- 结构化拖拽继续使用上游 `Drag`，仅增加指向当前目标父节点的虚线提示。
- 诊断记录仍默认脱敏，不自动上传。

## v0.5.9：诊断与回归模式

本版建立真实思源运行反馈链路，不增加新的导图业务功能：

- 顶部菜单和命令面板可打开“诊断与回归”。
- 支持运行存储、导图、检查点、数据完整性、设置和当前编辑器状态自检。
- 可开始或停止运行记录，捕获关键生命周期、操作、保存状态以及全局错误。
- 日志使用有界内存环形缓冲区，不会无限增长。
- 可导出 `yemind-diagnostics-日期时间.zip` 供问题分析。
- 默认只导出脱敏摘要，不包含导图标题、节点正文、批注、链接、图片或本地路径。
- 只有明确勾选后，诊断包才会附带完整导图内容，并在界面中给出隐私提醒。
- 所有诊断均在本机执行，不自动上传网络。

## 当前功能

- Dock：新建、打开、复制链接、重命名、删除导图
- 节点编辑、添加、删除、结构调整、撤销和重做
- 同一导图节点子树复制、剪切和粘贴
- 富文本、公式、模糊、备注、批注、待办、标签、图标、链接和图片
- 行内超链接、行内代码和代码块编辑器
- 概要、外框、关联线、多种布局和自动保存
- 搜索、替换、大纲和分屏
- 检查点与安全恢复
- 只读、禅模式和全屏
- 本地诊断、回归自检和脱敏诊断包导出

## 安装

把整个 `siyuan-yemind-zen` 文件夹放入：

```text
工作空间/data/plugins/siyuan-yemind-zen/
```

重启思源后启用 **YeMind Zen**。

## 开发原则

新增功能前先完整分析 `simple-mind-map` 对应插件、命令、事件、数据字段和关联模块。能够调用原生能力时不增加第二套状态层；一个版本只处理一个小主题。

后续版本计划见 `DEVELOPMENT_PLAN.md`。
