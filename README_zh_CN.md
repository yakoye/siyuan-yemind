# YeMind Zen

YeMind Zen 是一个基于 `simple-mind-map` 构建的思源笔记思维导图插件。

- 插件目录和插件 ID：`siyuan-yemind-zen`
- 思源显示名：`YeMind Zen`
- 当前版本：`0.6.6`

## v0.6.6：密度参数、极简样式面板与搜索打开

- 紧凑固定为左右 30、上下 2；默认继承上一版紧凑参数；舒展继承上一版默认参数。
- 整图“样式”面板缩窄到 220px，删除密度说明小字，间距、颜色和开关全部压缩。
- 思源全局搜索结果在鼠标按下阶段立即打开导图，避免搜索框失焦后结果先被宿主移除；节点定位继续作为打开后的增强动作。
- 适配视图改为四角聚焦 SVG 图标。

- 本地保存，不包含付费、试用、激活或会员逻辑

## v0.5.19：空节点删除、局部折叠与安全删除

- 空白非根大纲行在编辑状态下按 Backspace/Delete 会删除该行及对应节点，并把焦点移到相邻可见行。
- 大纲折叠状态改为编辑器局部 UID 集合，不再读写画布节点的 `expand` 数据；根节点和普通节点均可反复折叠、展开。
- 折叠按钮始终按稳定 UID 更新，叶节点保持不可点击占位；删除或失去子节点后自动清理陈旧折叠状态。
- 画布节点 Delete/Backspace 在捕获阶段和上游 `beforeShortcutRun` 双重拦截，Root 节点会被过滤，不再进入 KMind 的“无法删除所有 Root”错误弹窗。
- 大纲增加类似 VS Code indent-rainbow 的细层级引导线，四级颜色循环且不拦截鼠标事件。
- 节点结构、删除历史和持久化继续由 `simple-mind-map` 原生命令负责。

详细设计见 `docs/superpowers/specs/2026-07-18-v0.5.19-outline-delete-collapse-design.md`。

## v0.5.18：大纲焦点、整行拖拽与模糊显示重构

- 修复分屏/纯大纲一次 Delete 后退出编辑、Backspace 失效的根因：文字提交时不再把活动 Quill 行移入 Fragment 后重新插入。
- 增加同步提交重入保护；拖动和结构操作前只提交一次，避免非空节点被误写为空并显示为小圆圈。
- 删除六点拖拽把手；行空白区按住移动即可拖拽，正文内需长按后移动，保留正常光标定位和文字选择。
- 纵向决定前/子级/后，横向决定同级、成为子节点或向左降低层级；最终仍使用上游 MOVE/INSERT 命令和历史。
- 修复折叠后无法再次展开；叶节点改用不可点击对齐占位，折叠隐藏活动子节点时不会留下失效的焦点恢复任务。
- 公式按钮使用数学字体加粗 `π`；模糊改为画布和大纲统一的高斯/毛玻璃效果，悬停恢复。

详细设计与根因见 `docs/superpowers/specs/2026-07-18-outline-editing-drag-stability-design.md`，官方适配边界见 `docs/OFFICIAL_FEATURE_PARITY_v0.5.18.md`。

- 待办右键菜单统一为“添加待办/删除待办”，完成状态只由节点复选框切换。
- 用户界面将“挖空/取消挖空”改为“模糊/取消模糊”，公式按钮从 `Fx` 改为 `π`，底层旧数据保持兼容。
- 诊断日志增加问题时间标记、前后事件窗口、大纲编辑/IME/拖拽事务和版本一致性检查，并聚合高频视图噪声。
- 修复运行时版本常量长期停留在 `0.5.13` 的问题；清单、构建和诊断运行时现统一为 `0.5.17`。

## v0.5.13：节点拖动跟手性与旧连线修复

本版继续研究用户提供的 KMind Zen 0.33.0 官方运行源码，修复两项真实鼠标回归：

- 拖动开始后，隐藏父节点持有的旧“父 → 被拖节点”入线；多选拖动时分别处理每个顶层节点。

- 最终落点继续由上游 `MOVE_NODE_TO`、`INSERT_BEFORE`、`INSERT_AFTER` 执行，没有第二套结构、历史或保存实现。

## v0.5.11：启动恢复与可编辑大纲

- 插件先启动正式导图、设置和检查点仓库加载，再逐项注册 Tab、Dock、设置、命令和协议事件。
- 单个思源界面注册异常会被隔离并写入诊断，不再让 `ready` 假装已经完成、正式导图库却保持未加载。
- 启动后无需新建导图即可显示原有导图，恢复标签页等待真实仓库就绪后再判断导图是否存在。
- 分屏的大纲位于右侧，纯大纲占满工作区；两者共用同一套编辑逻辑。
- 大纲文字和结构修改继续调用上游命令，不直接改写树数据。

## v0.5.10：真实运行诊断修复

本版基于用户从思源 v3.7.2 导出的两份诊断包修复真实运行问题，不增加新的导图业务功能：

- 诊断生命周期自检改用独立临时仓库，不再把临时导图插入正式 Dock 后再删除。
- Dock 等待导图仓库加载完成后再显示最终列表。
- 恢复标签页等待容器真正可见且宽高有效后再创建导图编辑器。
- 纯大纲模式不再调用零尺寸导图画布的 `resize()`。
- 陈旧标签句柄会被清理，不能阻止同一导图重新打开。
- 待办复选框只在未完成和已完成之间切换；删除待办只保留在右键菜单。
- 结构化拖拽继续使用上游 `Drag`，仅增加指向当前目标父节点的虚线提示。
- 诊断记录仍默认脱敏，不自动上传。

## v0.5.9：诊断与回归模式

本版建立真实思源运行反馈链路，不增加新的导图业务功能：

- 顶部菜单和命令面板可打开“诊断与回归”。
- 支持运行存储、导图、检查点、数据完整性、设置和当前编辑器状态自检。
- 可开始或停止运行记录，捕获关键生命周期、操作、保存状态以及全局错误。
- 日志使用有界内存环形缓冲区，不会无限增长。
- 可导出 `yemind-diagnostics-日期时间.zip` 供问题分析。
- 默认只导出脱敏摘要，不包含导图标题、节点正文、批注、链接、图片或本地路径。
- 只有明确勾选后，诊断包才会附带完整导图内容，并在界面中给出隐私提醒。
- 所有诊断均在本机执行，不自动上传网络。

## 当前功能

- Dock：新建、打开、复制链接、重命名、删除导图
- 节点编辑、添加、删除、结构调整、撤销和重做
- 同一导图节点子树复制、剪切和粘贴
- 富文本、公式、模糊、备注、批注、待办、标签、图标、链接和图片
- 行内超链接、行内代码和代码块编辑器
- 概要、外框、关联线、多种布局和自动保存
- 搜索、替换、大纲和分屏
- 检查点与安全恢复
- 只读、禅模式和全屏
- 本地诊断、回归自检和脱敏诊断包导出

## 安装

把整个 `siyuan-yemind-zen` 文件夹放入：

```text
工作空间/data/plugins/siyuan-yemind-zen/
```

## 开发原则

新增功能前先完整分析 `simple-mind-map` 对应插件、命令、事件、数据字段和关联模块。能够调用原生能力时不增加第二套状态层；一个版本只处理一个小主题。

后续版本计划见 `DEVELOPMENT_PLAN.md`。
