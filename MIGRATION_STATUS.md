# YeMind Zen v0.6.5 迁移状态

## v0.6.5 当前状态

- 仓库文档版本仍为 `version: 1`；新增自定义间距只是 `projectStyle` 的可选字段，旧导图自动归一化为默认密度。
- 原 v0.6.4 `compact` 数据继续表示紧凑语义，但在新参数下更紧凑；原 `default` 数据保留默认枚举并采用旧紧凑参数。
- `comfortable` 不写死间距，而是继续继承当前主题原始间距。
- 搜索定位只增加标签页挂载期间的暂存 UID，不创建第二套标签页或节点导航状态。
- 节点颜色仍写入上游原生样式字段；统一色板只替换输入界面，不改变导图格式。
- 画布富文本可见性修复只同步当前节点真实文字色和安全编辑背景，不创建第二套富文本内容。
- v0.5.10 至 v0.6.4 的折叠、编辑、删除键、右键拖动画布、富文本、备注、批注、图片、节点样式、概要和全局搜索回归继续作为发布门禁。

## v0.6.4 当前状态

- 保持仓库版本 `version: 1`；旧导图没有 `projectStyle` 时自动补为继承主题的默认值。
- 全局搜索直接读取正式 MapRepository 并扩展思源搜索界面，不创建隐藏文档、影子节点或第二套索引。
- 叶子圆点与拖拽边界只影响大纲表现和 Pointer 路由，不改变导图结构。
- 整图样式只覆盖主题间距、彩虹线和背景色；节点局部样式字段保持不变。
- v0.5.10 至 v0.6.3 的折叠、编辑、删除键、右键拖动画布、富文本、备注、批注、图片、节点样式与概要回归继续作为发布门禁。

## v0.6.3 当前状态

- 本版不改变导图数据格式、检查点格式或仓库结构。
- 画布模式只更换表现层，现有 `canvasMode: select | pan` 设置继续兼容。
- 备注、批注、节点样式和菜单命令继续使用已有字段与上游命令，无需数据迁移。
- 悬停浮层位置为运行时几何计算，不写入导图。
- 节点菜单仅调整组织和图标，不改变快捷键、可用性判断或命令语义。
- v0.5.10 至 v0.6.2 的折叠、编辑、删除键、右键拖动画布、富文本、图片、备注、批注、节点样式与概要回归继续作为发布门禁。

## v0.6.2 当前状态

- 数据模型、地图持久化和检查点格式没有变化。
- 新安装默认画布模式从平移优先改为选（选择优先）；已有用户的显式设置继续保留。
- 旧工具栏动作没有删除命令能力，整理布局仍可通过右键菜单和命令使用。


## v0.6.1 当前状态

- 分屏和纯大纲的活动 Quill 表面现在拥有绝对的文字选择优先级；整行拖拽不会从编辑宿主、`.ql-editor` 或 contenteditable 后代启动。
- 顶部、左侧和底部工具栏已经按功能唯一性重新整理，检查点/历史能力迁移到左侧“历史”，底部只保留适配视图，顶部只保留搜索。
- 节点样式和线型成为顶部明确的图标+文字入口。
- 备注和批注使用内容内部关闭按钮；品牌和导图标题使用插件图标绿色。
- 既有 Root/分支折叠、数字/三角展开、编辑删除键、右键拖动画布、画布富文本和节点快捷控制均继续通过长期回归矩阵。

## 已完成

- 插件目录和插件 ID：`siyuan-yemind-zen`；思源显示名：`YeMind Zen`。
- 运行代码全部由 TypeScript 源码和 Vite 构建生成。
- `simple-mind-map` 作为实际编辑、拖拽、选择、布局、历史和节点剪贴板内核。
- 节点拖拽、框选、多选、复制粘贴、概要、关联线和外框均优先委托给上游原生插件与命令。
- 外框使用上游 `OuterFrame` 的 `groupId`、连续兄弟范围、渲染和文字编辑，不保存 YeMind Zen 私有几何数据。
- 导图仓库、设置写盘、标签页恢复和关闭前保存已经完成事务化与串行化保护。
- 已完成富文本、公式、模糊、行内链接、代码块、待办、长文备注、批注、搜索、替换、大纲、分屏和完整设置窗口。
- 检查点使用独立事务仓库，恢复前自动创建保护快照，并通过上游 `setFullData` 重建运行状态。
- 本地诊断服务使用独立临时仓库运行生命周期回归，不再触碰正式导图列表，并可导出默认脱敏诊断包，不自动上传。
- 思源恢复标签页会等待仓库就绪和容器非零尺寸；大纲模式不会向隐藏画布发送 resize。
- 待办复选框采用二态切换；结构化拖拽使用官方矩形/尾区/同级通道候选与贝塞尔提示，但最终移动仍由上游原生命令执行。
- 插件启动先加载正式仓库并隔离可选界面注册失败，已有导图无需通过新建操作触发延迟加载。
- 分屏和纯大纲已支持 IME 安全的事务式编辑，结构变化继续由上游原生命令和历史栈负责。
- 拖动时隐藏旧父节点入线，候选等待会逐帧自动稳定，混合布局按目标节点选择引导线方向。
- 主题和边线外观已改为官方源码驱动的适配层：默认弧线、13 个官方同名主题、思源明暗跟随、主题彩虹分支和每图持久化。
- 布局选择已开放当前 `simple-mind-map` 运行时的全部 14 种布局。
- 富文本浮层已局部挂载并隔离堆叠层，不再覆盖思源设置；节点编辑态文字继承当前节点颜色。
- 导图节点、分屏大纲和纯大纲共用选区富文本目标协议；大纲通过单活动 Quill 会话保留局部富文本。
- 文字/背景颜色使用 52 色板、默认重置、原生更多颜色和可编辑 HEX/RGB；输入事件不会串入节点正文。

- 分屏和纯大纲采用稳定 UID 行增量更新，活动 Quill、IME、选区和本地草稿不会因普通保存被销毁。
- 大纲每条可移动节点支持整行 Pointer 拖拽，最终仍使用上游 MOVE/INSERT 命令和历史。
- 备注支持长文、图片粘贴、窗口缩放与悬停预览；批注支持独立图标和悬停预览。
- 诊断支持问题时刻标记、事件窗口和清单/运行时/构建版本一致性检查。
- 大纲文字同步采用真正不移动活动行的 keyed patch；普通输入、Backspace/Delete 和自动保存不再破坏焦点。
- 大纲拖拽已移除六点把手，改为整行 Pointer 交互和横向层级意图；结构变更仍由上游命令和历史负责。
- 折叠/展开、叶节点占位、数学 π 和画布/大纲统一高斯模糊已完成。

- 空白非根大纲行支持 Backspace/Delete 结构删除，并恢复相邻可见行焦点。
- 大纲折叠已与画布节点 `expand` 解耦；Root/分支可反复展开，陈旧折叠 UID 自动清理。
- 画布删除在捕获阶段和上游快捷键钩子中双重过滤 Root，不再触发“无法删除所有 Root”弹窗。
- 大纲已加入不影响编辑和拖拽命中的循环层级引导线。
- Root 与普通分支现已共享原生 `expand` 状态；启动、画布、分屏和大纲不会再互相覆盖折叠结果。
- Root 与普通分支的折叠数字均为可点击展开入口；快捷 `+ / −` 只在节点选中且展开时显示。
- 折叠节点的展开判断现读取持久化 `nodeData.children`，Root、数字和大纲三角不会再因 live children 为空而失效。
- 大纲 Quill 的 Backspace/Delete 已与全局节点删除快捷键双层隔离；空节点结构删除仍为独立明确事务。
- pan/select 两种画布模式均支持右键平移，拖动使用抓手并消费松开后的菜单事件。
- 画布右键拖动现在在 window mouseup/blur 和文本编辑开始时可靠收尾；画布 Quill 选区不会再继承残留禁选状态。
- 真实 SVG 双击、局部选择、共享工具栏和局部格式化已纳入发布级集成回归。
