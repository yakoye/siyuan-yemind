# YeMind Zen v0.5.21 迁移状态

## 已完成

- 插件目录和插件 ID：`siyuan-yemind-zen`；思源显示名：`YeMind Zen`。
- 运行代码全部由 TypeScript 源码和 Vite 构建生成。
- `simple-mind-map` 作为实际编辑、拖拽、选择、布局、历史和节点剪贴板内核。
- 节点拖拽、框选、多选、复制粘贴、概要、关联线和外框均优先委托给上游原生插件与命令。
- 外框使用上游 `OuterFrame` 的 `groupId`、连续兄弟范围、渲染和文字编辑，不保存 YeMind Zen 私有几何数据。
- 导图仓库、设置写盘、标签页恢复和关闭前保存已经完成事务化与串行化保护。
- 已完成富文本、公式、模糊、行内链接、代码块、待办、长文备注、批注、搜索、替换、大纲、分屏和完整设置窗口。
- 检查点使用独立事务仓库，恢复前自动创建保护快照，并通过上游 `setFullData` 重建运行状态。
- 本地诊断服务使用独立临时仓库运行生命周期回归，不再触碰正式导图列表，并可导出默认脱敏诊断包，不自动上传。
- 思源恢复标签页会等待仓库就绪和容器非零尺寸；大纲模式不会向隐藏画布发送 resize。
- 待办复选框采用二态切换；结构化拖拽使用官方矩形/尾区/同级通道候选与贝塞尔提示，但最终移动仍由上游原生命令执行。
- 插件启动先加载正式仓库并隔离可选界面注册失败，已有导图无需通过新建操作触发延迟加载。
- 分屏和纯大纲已支持 IME 安全的事务式编辑，结构变化继续由上游原生命令和历史栈负责。
- 拖动时隐藏旧父节点入线，候选等待会逐帧自动稳定，混合布局按目标节点选择引导线方向。
- 主题和边线外观已改为官方源码驱动的适配层：默认弧线、13 个官方同名主题、思源明暗跟随、主题彩虹分支和每图持久化。
- 布局选择已开放当前 `simple-mind-map` 运行时的全部 14 种布局。
- 富文本浮层已局部挂载并隔离堆叠层，不再覆盖思源设置；节点编辑态文字继承当前节点颜色。
- 导图节点、分屏大纲和纯大纲共用选区富文本目标协议；大纲通过单活动 Quill 会话保留局部富文本。
- 文字/背景颜色使用 52 色板、默认重置、原生更多颜色和可编辑 HEX/RGB；输入事件不会串入节点正文。

- 分屏和纯大纲采用稳定 UID 行增量更新，活动 Quill、IME、选区和本地草稿不会因普通保存被销毁。
- 大纲每条可移动节点支持整行 Pointer 拖拽，最终仍使用上游 MOVE/INSERT 命令和历史。
- 备注支持长文、图片粘贴、窗口缩放与悬停预览；批注支持独立图标和悬停预览。
- 诊断支持问题时刻标记、事件窗口和清单/运行时/构建版本一致性检查。
- 大纲文字同步采用真正不移动活动行的 keyed patch；普通输入、Backspace/Delete 和自动保存不再破坏焦点。
- 大纲拖拽已移除六点把手，改为整行 Pointer 交互和横向层级意图；结构变更仍由上游命令和历史负责。
- 折叠/展开、叶节点占位、数学 π 和画布/大纲统一高斯模糊已完成。

- 空白非根大纲行支持 Backspace/Delete 结构删除，并恢复相邻可见行焦点。
- 大纲折叠已与画布节点 `expand` 解耦；Root/分支可反复展开，陈旧折叠 UID 自动清理。
- 画布删除在捕获阶段和上游快捷键钩子中双重过滤 Root，不再触发“无法删除所有 Root”弹窗。
- 大纲已加入不影响编辑和拖拽命中的循环层级引导线。
- Root 与普通分支现已共享原生 `expand` 状态；启动、画布、分屏和大纲不会再互相覆盖折叠结果。
- Root 与普通分支的折叠数字均为可点击展开入口；快捷 `+ / −` 只在节点选中且展开时显示。
